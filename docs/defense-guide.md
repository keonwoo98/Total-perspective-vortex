# 0부. 1분 요약

> 사람 머리에서 잰 전기 신호(**EEG**)만 보고, 그 사람이 **왼손/오른손(또는 손/발)을 움직이는 상상**을 하는지 컴퓨터가 맞히는 **뇌-컴퓨터 인터페이스(BCI)**.

흐름 한 줄:
**EEG 로드(MNE) → 7–30Hz 필터·2초 조각내기 → 직접 만든 CSP로 64채널을 핵심 4숫자로 압축 → LDA로 분류 → cross_val_score로 정직하게 채점 → 109명·6실험 평균 65.8%(≥60%) → 2초 내 스트림 예측.**

핵심 결과: **109명 6실험 평균 0.658 ≥ 0.60 통과** (PDF 기준 0.6261보다 높음). 보너스 5개(자작 고유분해·튜닝·자작 LDA·FBCSP·다른 데이터셋) 전부 구현.

# 2부. 개념 차근차근 (어려운 것까지 전부)

## 2.1 EEG = 머리에 붙인 마이크 64개

뇌 뉴런이 활동하면 미세 전기가 흐릅니다. 머리에 전극(=마이크)을 붙이면 그 전기의 흔들림을 잴 수 있어요 = **EEG(뇌전도)**.

- 마이크 **64개**(채널), 초당 **160번**(160Hz) 측정.
- 마이크 하나 = **숫자 한 줄**(시간마다 "지금 얼마나 출렁였나"). 점으로 이으면 **출렁이는 선**.
- 64개가 동시에 → **숫자 64줄 표**. 이게 EEG 데이터의 본체.
- 값은 **마이크로볼트(µV)** 수준 — 아주 작아요.

## 2.2 채널·전극 위치 (C3, Cz, C4)

전극 이름은 머리 위 위치예요(국제 10–10 규칙). 우리한테 중요한 건 **운동 영역** 세 개:
- **C3** = 왼쪽 뇌 (→ 오른손 담당)
- **C4** = 오른쪽 뇌 (→ 왼손 담당)
- **Cz** = 정수리 (→ 발)

뇌는 **반대쪽**을 제어해요(오른손은 왼쪽 뇌가). 이 좌우 교차가 핵심 단서가 됩니다.

## 2.3 비밀의 정체: ERD = "반대쪽이 조용해진다"

딱 3가지:
1. **좌우 교차** (2.2): 오른손 ↔ 왼쪽뇌(C3), 왼손 ↔ 오른쪽뇌(C4).
2. **뮤(mu, 8–12Hz)·베타(beta, 13–30Hz) 리듬**: 운동 영역은 쉴 때 이 주파수로 규칙적으로 진동(idle처럼).
3. **ERD(Event-Related Desynchronization, 사건관련 비동기화)**: 어떤 동작을 하거나 **상상만 해도**, 그쪽을 담당하는 영역의 뮤·베타 진동이 **약해진다(조용해진다)**.

→ **왼손 상상 → 오른쪽뇌(C4) ERD(조용) / 오른손 상상 → 왼쪽뇌(C3) ERD(조용).** 이게 1.2의 "비밀". 그래서 **7–30Hz(뮤+베타)만** 봅니다.

## 2.4 출렁임을 숫자로 = 분산(variance)

"조용하다/활발하다"를 컴퓨터가 다루려면 숫자가 필요해요.
- 한 채널이 위아래로 **크게** 출렁 → 큰 숫자, **잔잔** → 작은 숫자.
- 이 "출렁임 크기"를 재는 표준 숫자가 **분산(variance)**.
- **어떻게 숫자 하나가 되나**: 각 시점 값이 **평균에서 떨어진 거리를 제곱해 평균**낸 것.
  - 예) 값이 `+2,-2,+2,-2`(평균 0) → 제곱 `4,4,4,4`의 평균 = **4** (크게 출렁). 값이 `+0.1,-0.1,...` → 제곱 평균 ≈ **0.01** (잔잔).
  - **왜 제곱?** +/− 방향은 신경 안 쓰고 "얼마나 떨어졌나"만 보려고(부호 없애기).

→ "C4가 조용해졌다" = "C4의 분산이 작아졌다".

## 2.5 공분산(covariance) = "두 채널이 같이 출렁이나"

- **분산**: 한 채널이 **얼마나** 출렁이나.
- **공분산**: 두 채널이 **함께** 움직이나?
```
A: +2 -2 +2 -2
B: +2 -2 +2 -2  → A와 똑같이 → 공분산 큼(양수)
C: -2 +2 -2 +2  → 반대로     → 공분산 음수
D: +1 +1 -1 -1  → 딴짓       → 공분산 ≈ 0
```
- **공분산 행렬**: 64채널이면 **64×64 표**. 칸 (i,j) = "채널 i와 j가 같이 출렁이는 정도". 대각선 (i,i) = 채널 i의 **자기 분산**.
- 이 표 하나가 **"이 뇌 신호에서 채널들이 어떻게 엮여 출렁이나"**를 통째로 담아요. CSP의 재료.
- **`E @ E.T`가 왜 공분산?** `E`는 행=채널(64개), 열=시간(321). 행렬곱 결과의 `(i,j)`칸 = **E의 i번째 행 · j번째 행** = 채널 i의 321개 값과 채널 j의 321개 값을 **짝지어 곱한 뒤 다 더한 것**.
  - 두 채널이 같은 시점에 둘 다 크면(둘 다 +) 곱이 +로 쌓이고, 반대로 움직이면 −로 쌓여요 → 그 합이 곧 **"같이 움직인 정도(공분산)"**.
  - `E @ E.T` = 이 계산을 **64×64 모든 채널쌍에 대해 한 번에**. 대각선 `(i,i)` = 채널 i를 자기 자신과 = **자기 분산**.

## 2.6 차원 축소 & CSP — "똑똑한 믹싱 보드"

문제: 한 조각 = 64채널 × 321시간 ≈ **2만 숫자**. 너무 크고 잡음 많고, 사람마다 중요한 채널이 다름.

해결(**차원 축소**): 정답을 품은 **4숫자**로 압축. 방법 = **CSP**.

**믹싱 보드 비유**: DJ 콘솔의 슬라이더 64개로 마이크들을 섞어 **"왼손일 때 크게 출렁, 오른손일 때 잔잔해지는 슈퍼 마이크"**를 만든다. 그 슈퍼 마이크의 출렁임 하나만 봐도 왼/오른이 거의 보임.
- 컴퓨터가 **정답을 아는 예시 수백 개**를 보고, "두 손의 출렁임 차이를 가장 크게 벌리는 슬라이더 조합"을 자동 계산.
- 슈퍼 마이크 **4개**를 만들어 → 64줄을 **4숫자**로.

**CSP = Common Spatial Patterns(공통 공간 패턴).** '공간' = "어느 채널을 어떻게 섞나".

## 2.7 CSP 수학 (천천히, 빠짐없이)

서브젝트가 직접 구현하라는 부분. 4단계예요.

**(1) 클래스별 공분산 C1, C2**
- 왼주먹 조각들 → 평균 공분산 **C1** (64×64). 오른주먹 → **C2**.
- 의미: C1 = "왼주먹일 때 채널들이 전형적으로 어떻게 엮여 출렁이나"의 지문. C2 = 오른주먹 지문.
- **실측(피험자1)** 대각선 비교 — 2.3의 현상이 숫자로:
  ```
              왼주먹(C1)  오른주먹(C2)
  C3(왼뇌)     0.0082      0.0078    ← 오른주먹일 때 C3 조용
  C4(오른뇌)   0.0074      0.0087    ← 왼주먹일 때 C4 조용
  ```

**(2) 일반화 고유분해 `eigh(C1, C1+C2)`** ← 클라이맥스
- **먼저 쉬운 정의 (고유벡터/고유값이 뭔지)**: 2.6의 "슬라이더 조합 하나(=64채널을 섞는 방향 `w`)"가 바로 **고유벡터**예요. 그 조합에 매겨진 점수 `λ`가 **고유값**.
  - 고유벡터 = "행렬을 곱해도 **방향은 안 바뀌고 길이만 λ배** 되는 특별한 방향". 여기선 **두 손을 가장 잘 가르는 믹싱 방향**.
- 입력: C1과 C1+C2(전체). 출력: **후보 슈퍼마이크(=고유벡터) 64개 + 각각 점수 λ(0~1)**.
- **λ = "이 믹싱으로 거른 출렁임 중 왼주먹이 차지하는 비율"**:
  - λ→1: 왼주먹 전용(왼에 크게/오른에 작게), λ→0: 오른주먹 전용, **λ≈0.5: 반반=구분 못함**.
  - λ=0.5면 그 슈퍼마이크는 **왼/오른 모두 똑같이 출렁여** 구별에 도움이 안 돼요. λ=0.9면 왼주먹일 때 훨씬 크게 출렁여 한눈에 구별. **그래서 0.5에서 멀수록 좋음.** (전체 C1+C2로 나눠 "비율"이라 0~1.)
- **수식 평이하게**: `C1 w = λ (C1+C2) w` = "방향 `w`로 걸렀을 때 **왼주먹 출렁임(C1)이 전체 출렁임(C1+C2)의 딱 λ배**가 되는 `w`를 찾아라." 보통 고유값 문제는 `A w = λ w` 인데, 여기선 오른쪽이 `(C1+C2) w`라 **두 행렬을 동시에 보는 "일반화(generalized)" 버전**.
- **실측**: 64개 λ 중 최선이 0.378 / 0.619 — 살짝만 치우침. 왼/오른손은 원래 구분이 미묘해서(둘 다 손, 가까운 뇌) → exp0가 제일 어렵고 정확도 ~0.57.

**(3) 정렬·선택**
- 64개 후보를 **"λ가 0.5에서 먼 순서"**로 줄 세워(코드 `argsort(|λ-0.5|)[::-1]` = "0.5와의 거리를 작은→큰으로 정렬 후 뒤집기"), 가장 잘 가르는 **상위 4개** 믹싱(=고유벡터)만 골라 `filters_`(4×64)에 저장 = 슈퍼마이크 W.
  - (0.5에서 멀다 = 한쪽 손에 치우쳤다 = 잘 가른다. 0.378·0.619 같은 게 위로 가고, 0.5 근처는 버림.)

**(4) 특징 = log-분산 (transform)**
- 한 조각 E에: `Z = filters_ @ E` (4×321) = 슈퍼마이크 4개의 출력 신호.
- 각 출력의 **분산**(출렁임 크기) → 4숫자 → **비율로 정규화** → **log** 씌움.
- **비율로 정규화** = 4숫자를 합이 1이 되게 나눔. 조각마다 전체적으로 세게/약하게 출렁인 차이를 지우고 **순수한 4마이크 사이의 비율만** 남겨, 조각끼리 공정하게 비교.
- **왜 log?** 분산은 항상 양수이고 분포가 한쪽으로 쏠려서, log를 씌우면 펴져서 분류기(직선)가 가르기 좋아져요.
- **실측**: X(45,64,321) → F(45,**4**). 왼주먹 평균 `[-1.57,-1.08,-1.53,-1.55]`, 오른주먹 `[-1.32,-1.81,-1.27,-1.34]` — **둘째 숫자 -1.08 vs -1.81로 확연히 다름** → 정답 단서가 담김.

> (보너스에서 우리는 `eigh`를 **직접**(Jacobi 회전) 구현했어요 — 5부 A.)

## 2.8 PCA vs CSP (왜 CSP인가 = 지도 vs 비지도)

둘 다 "슈퍼마이크"로 차원을 줄이지만 **기준이 다름**:

| | PCA | CSP |
|---|---|---|
| 정답(라벨) 사용 | ❌ (**비지도 unsupervised**) | ✅ (**지도 supervised**) |
| 찾는 것 | 가장 큰 변화 | 두 클래스의 가장 큰 **차이** |
| 약점 | 큰 노이즈(눈 깜빡임)에 낚임 | (구분 전용) |

우리 일은 "왼/오른 **구별**"이고 그 차이는 "어느 쪽이 조용한가"라는 공간 패턴 → **CSP가 정확히 그걸 노리는 도구**. EEG BCI 표준이고 60% 달성에 유리.

## 2.9 전처리 vs 훈련 + 데이터 누수 (제일 중요한 방법론)

단계는 두 종류:
- **고정 전처리(배움 없음)**: 필터(7–30Hz), 에포크 자르기 — 누구에게나 같은 규칙. `fit` 없음.
- **훈련(데이터에서 배움)**: **CSP, LDA** — `fit`이 있음 = 데이터에서 배운다 = 모델의 일부.

→ **CSP는 전처리가 아니라 "훈련되는 모델"** (위치만 분류기 앞).

**데이터 누수(leakage) 방지**: CSP는 정답을 보고 배우므로, **시험 데이터까지 보면 컨닝**. 그래서 CSP를 **Pipeline 안에** 넣어 `cross_val_score`가 **fold마다 학습분으로만 CSP를 다시 배우게** 함. → 정직한 점수.

## 2.10 LDA = "울타리 긋기" (수학까지)

CSP가 만든 **4숫자 = 4차원 점**. 왼주먹 점들과 오른주먹 점들이 각각 뭉침. **LDA**가 그 사이에 **직선(울타리)**을 그어 가름.

```
   🔵🔵        ← 한 무리 (중심 μ0)
  🔵🔵 ╲
        ╲ 울타리 (방향 w에 수직)
       ╲ 🔴🔴  ← 다른 무리 (중심 μ1)
```
**수학 (4줄)**:
- `μ0, μ1` = 두 무리 중심(각 클래스 4숫자 평균).
- `Σ` = 무리 퍼짐(공분산). **2.5의 공분산 표와 똑같은 것인데, 이번엔 채널이 아니라 CSP가 만든 4숫자에 대한 4×4 표** — 점구름이 어느 방향으로 길쭉한지 담음.
- **`w = Σ⁻¹(μ1 − μ0)`** = 울타리 **방향** (= "중심→중심" 방향 `μ1−μ0`을, 무리 퍼짐 `Σ`로 보정).
  - **`Σ⁻¹`(역행렬) = "나누기의 행렬판".** 어떤 축으로 점이 많이 퍼져 있으면 그 방향 가중치를 **나눠서 줄여요** → "**많이 퍼진(노이즈 큰) 축은 덜 믿는다**". (숫자면 1/Σ, 행렬이면 Σ⁻¹.)
- `b = −w·(μ0+μ1)/2 + log(p1/p0)` = 울타리 **위치**(두 중심 가운데 + 클래스 균형 보정).
- 판정: **`w·x + b`의 부호** (양수→class1, 음수→class0).
  - **`w·x`(내적/dot product)** = 점 `x`(4숫자)를 방향 `w`에 **투영한 값** = 각 숫자끼리 곱해 다 더함(`w₁x₁+w₂x₂+w₃x₃+w₄x₄`). "울타리에서 어느 쪽으로 얼마나 떨어졌나"를 한 숫자로 만든 것.
- **실측(exp3)**: μ0 `[-1.73,-1.59,-1.08,-1.49]`, μ1 `[-1.17,-1.52,-1.96,-1.23]`, `w = [4.6,-6.07,-11.23,-2.04]` — **차이 큰 셋째 숫자(μ:-1.08 vs -1.96)에 큰 가중치(-11.23)**.

## 2.11 교차검증 cross_val_score (정직한 채점)

"이 모델 좋은가?"를 정직하게 재는 법:
- ❌ 45개로 학습하고 같은 45개로 시험 → 외운 거라 가짜 고득점.
- ✅ **일부를 숨기고** 나머지로 학습 → **숨긴 걸로 시험** → 돌려가며 반복 → 평균.
- 우리: `ShuffleSplit(10, test_size=0.2)` = **10라운드**, 매번 20%(≈9조각) 숨기고 80%로 학습 후 9개로 시험.
- Pipeline이라 **fold마다 CSP까지 재학습** → 누수 없음(2.9).
- 출력 예: `[0.78 0.78 0.89 ... 1.0 ... 0.67]` (각 9개 중 몇 개 = k/9) → 평균 = 실력.

## 2.12 sklearn / MNE & "transformer" 개념

- **MNE** = 뇌 신호 전문가: EDF 다운로드·필터·에포크.
- **scikit-learn** = 머신러닝 전문가: Pipeline·분류기·cross_val_score.
- 우리 CSP를 sklearn `BaseEstimator`+`TransformerMixin`을 상속시켜 만들면 → sklearn 기계(Pipeline 등)에 **그대로 꽂히는 부품**이 됨. `fit`(배우기)/`transform`(변환하기)을 갖춘 "변환기(transformer)".

## 3.1 데이터셋 구조 (전부 PhysioNet이 정해둔 것)

출처: **PhysioNet "EEG Motor Movement/Imagery"**(eegmmidb). 109명, 64채널, 160Hz, `.edf` 파일.

피험자마다 녹음 **14개(run)**, PhysioNet이 내용을 정해둠:

| run | 내용 |
|---|---|
| R01, R02 | 그냥 쉬기(baseline) — **안 씀** (그래서 우리 폴더에 없음) |
| **3, 7, 11** | 실제로 왼/오른 주먹 |
| **4, 8, 12** | 상상으로 왼/오른 주먹 |
| **5, 9, 13** | 실제로 양주먹/양발 |
| **6, 10, 14** | 상상으로 양주먹/양발 |

→ `config.py`의 `runs: [3,7,11]`은 "실제 주먹 녹음 3개"라는 뜻. 같은 과제를 3번 반복 녹음해서 번호가 흩어짐.

## 3.2 annotations: T0/T1/T2 = 녹음 안의 "동작 표시"

한 run은 ~2분 연속 녹음. 그동안 화면 지시가 **언제 시작됐는지** 표시가 박혀 있음(자막처럼):
- **T0** = 쉼, **T1** = 동작1 시작, **T2** = 동작2 시작.
- **T1/T2 뜻은 run마다 다름**: 주먹 run → T1=왼주먹, T2=오른주먹 / 손발 run → T1=양주먹, T2=양발. **(그래서 run별 라벨링이 필수.)**

## 3.3 "6개 실험" = 서로 다른 맞히기 게임 6종

4가지 과제(실제/상상 × 손좌우/손발)에서 **6개의 이분 게임**을 만듦:

| 실험 | run | 클래스0 vs 클래스1 | 109명 정확도 |
|---|---|---|---|
| exp0 | 3,7,11 | 왼주먹(T1) vs 오른주먹(T2) | 0.57 (제일 어려움) |
| exp1 | 4,8,12 | 상상 왼 vs 오른 | 0.58 |
| exp2 | 5,9,13 | 양주먹(T1) vs 양발(T2) | 0.69 |
| exp3 | 6,10,14 | 상상 손 vs 발 | 0.61 |
| exp4 | 3,7,11 vs 4,8,12 | 실제 vs 상상(손) | 0.73 |
| exp5 | 5,9,13 vs 6,10,14 | 실제 vs 상상(양측) | 0.76 |
| | | **6개 평균** | **0.658 ✅** |

- exp0~3 = 4과제 그대로(T1 vs T2). exp4~5 = "실제냐 상상이냐"(여러 run을 섞은 복합 — `run_all`에서만 평가).
- **`label` 0/1은 우리가 정하는 "A팀/B팀"** — 이게 게임을 정의함.
- 좌/우손이 낮고 손/발·실제/상상이 높은 건 뇌과학적으로 당연(가까운 영역 vs 먼 영역).

## 3.4 실제 데이터는 이렇게 생겼다 (피험자1, run3 실측)

- `Raw` = **자막 달린 영상**: ① 숫자 64줄(영상) + ② T0/T1/T2 자막.
- 신호 배열: **(64, 20000)** = 64채널 × 20000샘플 = 125초. 값 ~`-57µV`.
- 자막: `T0=15, T1=8, T2=7`개. 타임라인 `t=0 T0 → t=4.2s T2(오른주먹) → t=8.3 T0 → t=12.5s T1(왼주먹) → ...`.
- 데이터는 **레포에 없음**(코드만 제출). 첫 실행 시 `mne_data/`(프로젝트 폴더, ~3.1GB)로 자동 다운로드.

## 3.5 데이터 크기를 완전히 이해하기 (샘플·Hz·조각) ⭐ 제일 헷갈리는 부분

두 숫자 **20000**(원본)과 **321**(조각)이 서로 다른 것임을 구분하면 끝입니다.

**(1) "샘플"과 "Hz"가 뭔지**
- **샘플(sample)** = 한 순간의 측정값 1개 (마이크가 "찰칵" 하고 값 하나 찍기).
- **160Hz** = 1초에 **160번** 찰칵 = **초당 160샘플**.
- → **시간(초) × 160 = 샘플 수.** 예: `125초 × 160 = 20000샘플`. 거꾸로 `20000 ÷ 160 = 125초`.

**(2) 한 run의 크기 (데이터셋이 주는 원본)**
- 1 run = **64채널 × 20000샘플(=125초)**. 매 "찰칵"마다 64채널 값이 동시에 찍힘.
- 한 사람 = task run 12개 → `64 × 20000 × 12` (≈ 1540만 개 숫자).
- 데이터셋 전체 = `64 × 20000 × 12 × 109명`.

**(3) 왜 전체를 안 쓰고 "자르나"**
- 125초 run엔 `쉼 → 동작 → 쉼 → 동작 ...`이 섞여 있어요. 우리가 원하는 건 **"한 번의 동작에 해당하는 신호"**.
- 그래서 동작 시작 시각(T1/T2)부터 **2초만** 오려서 "한 번의 시도" = 한 **조각(epoch)**으로 만듦.

**(4) 그래서 321의 정체**
- 한 조각 = 2초 = `2 × 160 = 320샘플`, 양 끝(0초·2.0초)을 다 포함해서 **+1 = 321샘플**.
- → 조각 하나 = **64채널 × 321샘플**. (`config`의 `TMIN=0, TMAX=2.0, SFREQ=160`에서 나옴.)

**(5) 두 숫자의 관계 한 그림**
```
원본:  64 × 20000   (한 run, 125초 연속 녹음)
  │   동작 표시(T1/T2)마다 2초씩 오림 (한 run에 약 15번)
조각:  64 × 321   짜리 약 15개          ← 321은 여기서 나옴
  │   3개 run 모음
실험:  (45, 64, 321)  = 조각 약 45개
```
> **핵심: 20000 = "녹음 전체 길이(125초)", 321 = "동작 한 번을 2초 오려낸 길이".** 같은 데이터를 보는 **단위**가 다를 뿐.

# 4부. 코드 한 바퀴 (파일별, 실제 출력과 함께)

빌드 순서 = 데이터 흐름 순서. 아래로 갈수록 위 모듈을 재료로 씀.

## 4.1 `tpv/config.py` — 설정 + 실험표 (로직 없음, stdlib만)
- 상수: `SFREQ=160, FMIN/FMAX=7/30, TMIN/TMAX=0/2, N_COMPONENTS=4, REG=0.01, N_CHANNELS=64, N_SUBJECTS=109`.
- `DATA_DIR` = 데이터 캐시 위치(프로젝트 폴더, `__file__` 기준이라 폴더 복사 시 따라옴).
- **`EXPERIMENTS`** = 6실험을 **데이터(dict)로** 표현. 한 실험 = 클래스2개 명세 리스트, 명세 = `{runs, annotations, label}`. preprocessing이 이 dict만 읽어 어떤 실험이든 같은 코드로 처리.
- `RUN_TO_EXPERIMENT`: run→exp0~3 (단일 run CLI용).
- `get_seed()`: `TPV_SEED` 있으면 그 숫자(재현), 없으면 None(매번 다른 분할 — 서브젝트 요구).

## 4.2 `tpv/data.py` — `.edf` 받아 읽고 합치기
`load_raw(subject, runs)`:
1. `eegbci.load_data(subject, runs, path=DATA_DIR)` — 요청한 run만 다운로드/찾기.
2. `read_raw_edf` — 파일을 `Raw`로 읽기.
3. `eegbci.standardize` — 채널 이름 청소(`Fc5.`→`FC5`).
4. `concatenate_raws` — 여러 run을 하나로 이어붙임.
5. `set_montage("standard_1005")` — 전극의 머리 위 3D 위치 부착.
6. sfreq 가드 — 160Hz 아니면 resample(이상 피험자 대비).
→ 결과: **자막 달린 영상(Raw)**.

## 4.3 `tpv/preprocessing.py` — 필터 + 2초 조각 (재료 만들기)
- **`filter_raw`**: ① `set_eeg_reference("average")` ② `filter(7,30, fir_design="firwin")`.
  - **reference(기준 전극)**: EEG는 절대값이 아니라 늘 "어떤 기준점 대비 전압". **average reference** = 기준점을 모든 채널 평균으로 잡아 **공통 잡음 제거**(필터와는 다른 축의 잡음).
  - **FIR 필터** = 특정 주파수만 통과시키는 디지털 "체"; 여기선 7–30Hz만 통과. **firwin** = 그 체를 **신호를 시간상 안 밀게** 설계하는 방식(2초 조각 위치가 안 어긋나게).
- **`epochs_from_raw(raw, annotations, label)`**: ① `events_from_annotations`(자막을 `[샘플번호,0,코드]`로) ② 원하는 표시만 고름(T0 버림) ③ `mne.Epochs(tmin=0, tmax=2.0)`로 각 표시에서 2초 오림 → X(개수,64,**321**), y=label로 채움. (321 = 2초×160 + 1.)
- **`build_dataset(subject, experiment)`**: `EXPERIMENTS`의 명세대로 run-group 로드·필터하고, 명세별로 오려 라벨 붙여 합침.
- **실측**: exp0 → X(45,64,321), 왼23/오22. (자막 T1=8 → 조각 8개처럼, 표시 하나당 조각 하나.)

## 4.4 `tpv/csp.py` — 심장 (2.7의 코드판)
- `_class_cov(epochs)`: `E@E.T` → 트레이스정규화·평균·대칭화·shrinkage → 클래스 공분산 C(64×64).
  - **트레이스(trace)** = 대각선 합 = 전체 출렁임 총량. 트레이스로 나눔 = 조각마다 전체 크기를 1로 맞춰 큰 조각이 평균을 독차지 못 하게.
  - **대칭화** = 반올림 오차로 표의 (i,j)≠(j,i)가 된 걸 평균 내 맞추기(공분산 표는 원래 대칭).
  - **shrinkage(수축, reg=0.01)** = 추정한 표를 단위행렬 쪽으로 1% 당겨, 데이터가 적을 때 역행렬이 폭주 안 하게 하는 보험.
- `fit(X,y)`: `y`로 클래스 나눔 → C1,C2 → `eigh(c1, c1+c2)`(또는 보너스 jacobi) → `|λ-0.5|` 정렬 → `filters_`(4×64).
- `transform(X)`: `filters_@E` → 분산 → 비율 → log → 4숫자.
- `solver="eigh"`(기본) / `"jacobi"`(보너스). `solver`는 `fit`에서만 읽음(옛 모델 호환).

## 4.5 `tpv/pipeline.py` — 조립 라인 공장
- `build_pipeline(csp, clf, ...)` → `Pipeline([("csp", CSP), ("clf", LDA)])`.
- `csp`: `scratch`(우리 CSP)/`mne`(검증용)/`fbcsp`(보너스). `clf`: `lda`/`svm`/`logreg`/`rf`/`own-lda`(보너스). **기본 scratch+lda**.
- **CSP가 첫 단계**라 cross_val_score가 fold마다 CSP 재학습 → 누수 방지.

## 4.6 `tpv/train.py` — 채점 + 학습·저장 (평가표 "Train")
1. **`cross_val_score`**(전체 파이프라인, 10-fold) → `[10개 점수]` + 평균 출력.
2. `train_test_split`로 80/20 → 80%로 `fit` → **`{pipeline, X_test, y_test, meta}`**를 `models/subjNNN_runRR_expE.joblib`로 저장(20% 9조각을 predict가 재생).
- **실측 `train(1,14)`**: `[0.7778 0.7778 0.8889 ... 1.0 ... 0.6667 0.8889]`, mean **0.8444**, X_test (9,64,321).

## 4.7 `tpv/predict.py` — 스트리밍 예측 (평가표 "Predict/Realtime")
- `joblib.load`(무거운 일, 루프 밖) → 9조각을 **하나씩**(스트림 흉내) `pipe.predict` → **2초 미만 assert** → `epoch NN: [pred] [truth] equal?` + `Accuracy`.
- **실측 `predict(1,14)`**: 9 epochs, **Accuracy 0.7778(7/9)**, 칩당 **0.28ms**(예산 2000ms). `[2]/[1]`은 내부 0/1을 1/2로 표시.

## 4.8 `tpv/evaluate.py` — 109×6 채점 + 60% 게이트 (평가표 "Score")
- `cross_val_accuracy(subj, exp)` = 한 셀 점수 1개.
- `run_all` = 6실험 × 피험자 이중 루프 → 실험별 평균 → **6개 평균의 평균 = grand mean**. `except`는 데이터/IO 실패만 skip.
- `compare_classifiers(subj, exp)` = 4분류기(lda/svm/logreg/rf) 비교 — CLI엔 없고 Python에서 `from tpv.evaluate import compare_classifiers`로 호출. `tune` = 보너스.
- **실측(3명)**: exp0 0.54…exp5 0.74, grand **0.7256**. **전체 109명 = 0.658**.

## 4.9 `mybci.py` — 입구 (CLI)
- `main(argv)`이 인자를 읽어 **dispatch**(라우팅만, 로직 없음):
  - 인자 0개 → `run_all()`
  - `<subj> <run> train|predict|tune` → 해당 함수.
- 검증: subject 1–109, run 3–14, mode 확인. 틀리면 usage + exit 2.

## 4.10 `tpv/viz.py` — 시각화 (평가표 Plot)
- `plot_raw_before_after`: raw vs 7–30Hz 필터 신호. `plot_psd_before_after`: **PSD**(Power Spectral Density = **주파수별 세기 그래프**; 가로축 주파수·세로축 그 주파수 세기) 필터 전/후 — 30Hz 위가 뚝 떨어진 게 보임. `plot_csp_patterns`: CSP 첫 슈퍼마이크를 **두피 지도(topomap)**로 그려 C3/Cz/C4 운동영역에 가중치가 몰린 걸 보여줌(2.2–2.3의 가장 직접적 시각 증거).
- `scripts/visualize.py`는 raw/PSD 쌍을 띄움. `plot_csp_patterns`는 Python에서 `from tpv import viz; viz.plot_csp_patterns(1, 3)`로 호출.

## 4.11 테스트 (무엇을 검증하나)

## A. 자작 고유분해 `tpv/jacobi.py` (Implementations) ⭐
- 목표: `scipy.linalg.eigh`를 **라이브러리 없이 직접**.
- **`jacobi_eigh(A)`**: 대칭 행렬을 **"회전"을 거듭해** 푸는 법. 비대각 칸을 회전으로 하나씩 **0으로** 만들어가요. **sweep(스윕)** = 표의 비대각 칸을 한 바퀴 도는 것; 여러 바퀴 돌면 비대각이 점점 0 → **대각선만 남고 = 고유값**, 누적한 회전 = 고유벡터. (비유: 비스듬히 누운 행렬을 회전을 거듭해 "똑바로" 세우기. 똑바로 서면 대각선이 답.)
- **`generalized_eigh(C1, B)`**: CSP의 `eigh(C1, C1+C2)`를 직접 푸는데, **whitening(백색화)**으로 표준 문제로 바꿔 `jacobi_eigh`를 두 번 호출.
  - **whitening = 데이터가 방향마다 길쭉/납작하게 찌그러진 걸, 모든 방향으로 똑같이 동그랗게 펴 주는 좌표 변환.** (모든 방향이 똑같아진 상태가 "백색 잡음(white noise)"을 닮아 '화이트'.)
  - 왜 필요? 일반화 문제 `C1 w = λ(C1+C2)w`는 `(C1+C2)` 때문에 좌표가 비뚤어져 **바로 못 풀어요.** 먼저 `(C1+C2)` 기준으로 좌표를 동그랗게 펴면, 남는 건 **평범한 고유값 문제 하나**라 `jacobi_eigh`로 바로 풀려요. (코드 `P=(U/√d).T`가 각 축을 그 퍼짐 √(고유값)으로 나눠 펴는 부분.)
- 검증: scipy와 **소수점 ~14자리(≈5e-14)까지 일치** = 사실상 완벽. `MyCSP(solver="jacobi")`. **실측: 실데이터에서 eigh와 0.8444로 동일.**

## C. 하이퍼파라미터 튜닝 `evaluate.tune` (Implementations)
- `GridSearchCV`로 `csp__n_components ∈ {4,6,8}`를 **누수 없이**(파이프라인 통째 inner CV) 탐색. `python mybci.py <s> <r> tune`. **실측: 최적 nc=8, 0.8444.**

## D. 자작 LDA `tpv/own_lda.py` (Implementations)
- 2.10의 수학을 numpy로 직접: `w=Σ⁻¹(μ1-μ0)`, `b`, `predict=sign(w·x+b)`. `build_pipeline(clf="own-lda")`. **실측: 0.7889** (sklearn LDA와 패리티). run_all 기본엔 미포함.

## F. FBCSP `tpv/fbcsp.py` (Feature engineering)
- 7–30Hz를 **4개 밴드(8-12/12-16/16-20/20-30)**로 쪼개 **밴드마다 CSP** → 8숫자. 뮤·베타를 따로 다룸. `build_pipeline(csp="fbcsp")`. **실측: 0.8222**(항상 일반 CSP보다 낫진 않음 — 더 정교한 방법 시연).

## G. 다른 데이터셋 `tpv/external.py` (Datasets)
- **BCI Competition IV-2a**(moabb)를 우리 `(X,y)` 형식으로. **실측: X ≈ (288, 22, ~1000)** — 22채널·250Hz·288 trial은 데이터셋 고정값(시간 길이는 moabb 기본 epoch 창에 따름). 모양이 달라도 우리 파이프라인 그대로 **≈0.85** (MyCSP가 채널 수 무관). PhysioNet과 **별개 실행**.

**핵심 숫자**

**자주 막히는 포인트**
- "run 숫자·T1/T2는 우리가 정한 게 아니라 **데이터셋이 정함**" (3.1~3.2).
- "label(0/1)은 **우리가 정하는** A팀/B팀" (3.3).
