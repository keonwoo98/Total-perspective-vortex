# 1부. 개념을 하나씩 (제일 쉬운 버전)

## 1-0. 한 문장

> **사람 머리에서 잰 전기 신호(EEG)만 보고, 그 사람이 "왼손을 움직이는 상상"을 하는지 "오른손"인지 컴퓨터가 알아맞히게 한다.**

이게 **BCI(뇌-컴퓨터 인터페이스)**입니다. 키보드 대신 "생각"으로 신호를 주는 것. 제목 "Plug your brain to the shell(뇌를 셸에 꽂아라)"의 뜻입니다.

## 1-1. EEG = 머리에 붙인 마이크 64개

뇌가 활동하면 미세한 전기가 흐릅니다. 머리에 전극(=마이크)을 붙이면 그 전기의 흔들림을 잴 수 있어요. 이게 **EEG**.

- 마이크 **64개**(채널), 초당 **160번**(160Hz) 측정.
- 실험 한 번(trial) = `(64채널 × 시간)` 2차원 배열. 마이크 64개가 동시에 녹음한 것.

## 1-2. 비밀: 생각한 손의 "반대쪽"이 조용해진다 (왜 가능한가)

딱 3가지만 알면 됩니다:
1. **좌우 교차**: 오른손은 **왼쪽** 뇌(전극 C3 근처), 왼손은 **오른쪽** 뇌(C4 근처)가 담당. 발은 정수리.
2. **뮤·베타 리듬**: 쉴 때 운동 영역은 8~30Hz로 규칙적으로 진동.
3. **ERD**: 어떤 동작을 하거나 **상상만 해도**, 그쪽 영역의 진동이 **약해진다(조용해진다)**.

→ **왼손 상상 → 오른쪽 머리가 조용 / 오른손 상상 → 왼쪽 머리가 조용.** "어느 쪽이 조용해졌나"가 정답 단서.

## 1-3. "조용함"을 컴퓨터가 숫자로 잡는 법

마이크가 녹음하는 건 **위아래로 출렁이는 선**입니다.
- 활발하면 → 크게 출렁 / 조용하면 → 잔잔.
- 이 **"출렁임의 크기"를 숫자 하나**로 잴 수 있어요 (큰 출렁=큰 숫자). 이 숫자를 **분산(variance)**이라고 부릅니다.

→ 컴퓨터는 왼쪽/오른쪽 마이크의 출렁임 숫자를 비교해서 "오른쪽이 작아졌네 → 왼손!" 이런 식으로 판단.

## 1-4. CSP = 똑똑한 "믹싱 보드" ⭐ (프로젝트의 심장)

현실은 지저분합니다: 정답 단서가 여러 마이크에 퍼져 있고, 잡음도 많고, 사람마다 중요한 마이크가 다릅니다.

그래서 **DJ 믹싱 보드**처럼, 64개 마이크를 똑똑하게 섞어서 **"왼손일 때 크게 출렁, 오른손일 때 잔잔해지는 슈퍼 마이크"**를 만듭니다.

- 그 슈퍼 마이크의 출렁임 숫자 하나만 봐도 왼손/오른손이 거의 보임.
- **어떻게 그 믹싱을 찾냐?** 정답을 아는 예시 수백 개를 보고, 컴퓨터가 "두 손의 출렁임 차이를 가장 크게 벌리는 슬라이더 조합"을 자동 계산.
- 슈퍼 마이크를 **4개** 만들어 → 지저분한 **64줄 신호를 깔끔한 4개 숫자**로 압축.

**이게 CSP(Common Spatial Patterns, 공통 공간 패턴)**. 서브젝트가 "직접 구현하라"는 차원 축소가 바로 이 부분.

**수학은 이만큼만**:
- 클래스별 **공분산 행렬**(채널들이 함께 어떻게 출렁이나, 64×64) 계산.
- 두 공분산의 차이를 최대화하는 방향을 **일반화 고유분해**(`scipy.linalg.eigh(C1, C1+C2)`)로 찾음.
- 고유값이 **1에 가까우면 클래스 A 전용 필터, 0이면 B 전용**. 0.5에서 먼 것부터 4개 선택.
- 특징 = 각 필터로 거른 신호의 **log(분산)**.

## 1-5. PCA vs CSP — 왜 CSP를 골랐나 (지도 vs 비지도)

둘 다 "슈퍼 마이크"로 차원을 줄이지만, **무엇을 기준으로 믹싱을 찾느냐**가 다릅니다.

| | PCA | CSP |
|---|---|---|
| 정답(라벨) 사용? | ❌ 안 씀 (**비지도, unsupervised**) | ✅ 씀 (**지도, supervised**) |
| 찾는 것 | 가장 크게 변하는 방향 | 두 클래스의 가장 큰 **차이** |
| 약점 | 큰 노이즈(눈 깜빡임)에 낚일 수 있음 | (두 클래스 구분 전용) |

우리 일은 "왼손 vs 오른손 **구별**"이고, 그 차이는 "어느 쪽 머리가 조용해졌나"라는 공간 패턴 → **CSP가 정확히 그걸 잡으라고 만들어진 도구**라서 선택. EEG BCI의 사실상 표준이고 60% 달성에 유리.

## 1-6. 전처리 vs 훈련 (헷갈리기 쉬운 핵심) + 데이터 누수

단계는 **두 종류**입니다:
- **고정 전처리(배움 없음)**: 필터(7–30Hz), 에포크 자르기 — 누구에게나 똑같은 규칙. `fit` 없음.
- **훈련(데이터에서 배움)**: **CSP**, **LDA** — `fit`이 있음 = 데이터에서 배운다 = 모델의 일부.

→ **CSP는 전처리가 아니라 "훈련되는 모델"**입니다(위치만 분류기 앞).

**데이터 누수(leakage) 방지** — 제일 중요한 방법론:
CSP는 정답을 보고 배우므로, **시험 데이터까지 보면 컨닝**이 됩니다. 그래서 CSP를 **Pipeline 안에 넣어** `cross_val_score`가 매 fold의 학습분으로만 CSP를 다시 배우게 합니다.

## 1-7. LDA = "울타리 치기"

이제 한 trial = 숫자 4개. **LDA(선형판별분석)**는 그 4숫자 공간에서 **왼손 무리와 오른손 무리를 가장 잘 가르는 직선(울타리)**을 긋고, 새 점이 울타리 어느 쪽이냐로 결정합니다. CSP가 일을 잘 해놔서 단순한 직선으로 충분.

## 1-8. "6개의 실험" = 서로 다른 맞히기 게임 6종

참가자들은 **4가지 과제**를 했어요 (실제/상상 × 손좌우/손발):

| | 실제로 움직임 | 상상만 함 |
|---|---|---|
| 왼주먹 ↔ 오른주먹 | 과제 A | 과제 B |
| 양주먹 ↔ 양발 | 과제 C | 과제 D |

여기서 **6개의 이분(A vs B) 게임 = 6개 실험**을 만듭니다:

| 실험 | 사용 run | 무엇을 구별 |
|---|---|---|
| exp0 | 3,7,11 | 실제 왼손 vs 오른손 |
| exp1 | 4,8,12 | 상상 왼손 vs 오른손 |
| exp2 | 5,9,13 | 실제 손 vs 발 |
| exp3 | 6,10,14 | 상상 손 vs 발 |
| exp4 | 3,7,11 vs 4,8,12 | 실제 vs 상상 (손) |
| exp5 | 5,9,13 vs 6,10,14 | 실제 vs 상상 (양측) |

- exp0~3 = 4과제 그대로(각 과제 안 T1 vs T2). exp4~5 = "실제냐 상상이냐"를 묻는 조합.
- "run"은 그냥 녹음 세션 번호(각 과제를 3번씩 녹음). 최종 점수 = **6게임 평균**.
- 우리 109명 결과: exp0 0.57, exp1 0.58, exp2 0.69, exp3 0.61, exp4 0.73, exp5 0.76 → **평균 0.658** (좌우손이 제일 어렵고, 손/발·실제/상상은 쉬움 — 문헌과 일치).

## 1-9. 데이터는 어디서 오나

- 출처: **PhysioNet "EEG Motor Movement/Imagery"** (eegmmidb). 109명, 64ch, 160Hz, `.edf` 파일.
- 코드(`tpv/data.py`)가 MNE로 **자동 다운로드** → **프로젝트 폴더의 `mne_data/`**(약 3.1GB)에 저장. **git엔 안 올라감**(코드만 제출).
- 파일 이름 예: `S006R03.edf` = Subject 006, Run 03.

## 1-10. sklearn / MNE 역할 분담

- **MNE** = 뇌 신호 전문가 🧠: EDF 다운로드·읽기, 7–30Hz 필터, 에포크 자르기, 전극 위치.
- **scikit-learn(sklearn)** = 머신러닝 전문가 🤖: Pipeline, 분류기(LDA…), `cross_val_score`, train/test 분할.
- 다리: 우리 CSP가 sklearn의 `BaseEstimator`+`TransformerMixin`을 상속 → sklearn 기계에 그대로 끼움.

## 1-11. 전체 흐름 한 장

```
PhysioNet (.edf)
  └ data.py:        MNE로 다운로드·읽기 → 64채널 raw
     └ preprocessing.py:  7-30Hz 필터(고정) + 2초 에포크로 자르기 → X(약45, 64, 321), y∈{0,1}
        └ csp.py(MyCSP):  [훈련] 슈퍼마이크 믹싱 배움 → 64줄을 4숫자로 (라벨 사용)
           └ LDA:         [훈련] 4숫자 공간에서 왼손/오른손 울타리 긋기
              └ cross_val_score:  컨닝(누수) 없이 채점 → 109명×6실험 평균 60% 이상 (우리 0.658)
              └ predict:   보류 데이터를 스트림처럼 재생, 2초 내 예측
```

# 4부. 보너스 (쉽게)

평가표 보너스 3박스를 모두 채웠습니다. 전부 **opt-in**(필수 경로는 그대로).

- **A. 자작 고유분해 (`tpv/jacobi.py`)** — scipy의 `eigh`를 **직접 만든 Jacobi 회전 알고리즘**으로 대체. "직접 구현" 핵심. scipy와 **1e-14로 일치** 검증. 사용: `MyCSP(solver="jacobi")`. → *Implementations*
- **C. 하이퍼파라미터 튜닝 (`evaluate.tune`)** — `GridSearchCV`로 n_components(4/6/8)를 **누수 없이** 자동 탐색. 사용: `python mybci.py 4 14 tune`. → *Implementations*
- **D. 자작 LDA (`tpv/own_lda.py`)** — 분류기를 numpy로 직접(클래스 평균·공분산·`w=Σ⁻¹(μ1-μ0)`). sklearn LDA와 패리티. 사용: `build_pipeline(clf="own-lda")`. → *Implementations*
- **F. FBCSP (`tpv/fbcsp.py`)** — 7–30Hz를 **4개 작은 밴드로 쪼개 밴드마다 CSP** → 특징 8개. "특징공학 + 더 복잡한 차원축소". 사용: `build_pipeline(csp="fbcsp")`. → *Feature engineering*
- **G. 다른 데이터셋 (`tpv/external.py`)** — **BCI Competition IV-2a**(moabb)로 좌/우손 분류. 다른 실험실·장비(22ch/250Hz)인데도 우리 파이프라인 그대로 동작, **정확도 0.85**. moabb는 `requirements.txt`에 포함(설치해도 핵심 라이브러리 버전 안 바뀜 → 필수 무손상). → *Datasets*

**보너스 한 방 시연**: `python scripts/bonus_demo.py` (5개 전부, G 포함). moabb는 `requirements.txt`에 포함돼 별도 설치·별도 venv 불필요 — `pip install -r requirements.txt` 한 번이면 전부 동작.

# 5부. 예상 Q&A 치트시트

| 질문 | 한 줄 답 |
|---|---|
| CSP가 뭐냐 | 두 클래스 분산 차이를 최대화하는 공간필터를 학습해 64채널→4숫자로 압축 |
| PCA 대신 CSP 왜? | PCA는 라벨 안 써서 노이즈에 낚일 수 있음; CSP는 라벨로 클래스 차이를 직접 노림 (지도 vs 비지도) |
| 왜 Pipeline 안에 CSP? | 누수 방지 — fold마다 학습분으로만 CSP 재학습 |
| CSP는 전처리야 훈련이야? | 훈련(모델의 일부). `fit`이 있고 데이터에서 배움. 필터만 고정 전처리 |
| 왜 7–30Hz? | 뮤·베타 감각운동 리듬(ERD 일어나는 대역) |
| W가 뭐냐 (WᵀX=X_CSP) | 공간필터 행렬(4×64), 각 trial에 적용. ch×time 평탄화 행렬 만드는 게 아님 |
| 2초 실시간 어떻게? | 무거운 로드는 루프 밖, 칩당 예측 1ms 미만, assert로 보장 |
| 60% 못 넘는 실험은? | 게이트는 6실험 평균(65.8%)에 적용. 좌우손이 원래 어려움 |
| 라이브러리 CSP 썼나? | 아니오. numpy 공분산 + scipy 고유분해로 직접. mne CSP와 1.1% 내 일치로 검증 |
| 고유값이 왜 [0,1]? | C1을 C1+C2로 정규화한 일반화 고유문제라 분산 비율이 0~1 |
| 데이터 어디 있나 | `mne_data/`(프로젝트 폴더, 3.1GB). git엔 코드만 |
| (보너스) 직접 만든 게 뭐냐 | 고유분해(Jacobi), 분류기(LDA), 차원축소(CSP/FBCSP) 전부 자작 + 튜닝 + 다른 데이터셋 |

**핵심 한 줄 요약**: 머리 잡음 듣고 맞히는 게임 → 생각한 손의 반대쪽이 조용해진다 → 출렁임을 숫자로 → CSP로 똑똑한 4숫자 압축(직접 구현) → LDA로 울타리 → 누수 없이 검증해 109명 평균 **65.8%**.
