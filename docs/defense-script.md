# 2부. 핵심 개념 완전 복습 (쉽고 세세하게)

## 2.1 EEG = 머리에 붙인 마이크 64개

- 머리에 전극(마이크) **64개**를 붙이고, 각자 **초당 160번** 전압을 잽니다. 단위는 **마이크로볼트(µV)**.
- 한 전극 = 숫자 한 줄. 점으로 이으면 **출렁이는 선**. 64개가 동시에 = **64줄짜리 표**.
- 한 사람의 한 녹음(run)은 약 125초 = **20000칸**(125초 × 160) → 표 모양 **(64, 20000)**.

## 2.2 채널과 C3/C4 (그냥 사실)

- 전극 이름 = 머리 위치. 앞글자=구역(F이마, **C정수리=운동구역**, P뒤, O뒤통수, T옆), 뒷숫자=좌/우(**홀수=왼쪽, 짝수=오른쪽, z=중앙**).
- **C3**(9번째 채널)=왼쪽 운동구역 → **오른손 담당**. **C4**(13번째)=오른쪽 운동구역 → **왼손 담당**. **뇌는 반대쪽을 제어**(좌우 교차).
- 64채널 중 **운동구역(C로 시작: C5 C3 C1 Cz C2 C4 C6)**이 손 동작의 핵심. 나머지 57개(이마·뒤통수 등)도 있지만 손 신호는 약함.
- **왜 C3/C4 2개만 안 쓰나?** 문제가 6종인데 **문제마다 정답이 숨은 채널이 다름**(손좌우=C3/C4, 발=Cz쪽). 그래서 채널을 손으로 못 박고, **CSP가 문제마다 최적 조합을 자동으로** 찾게 함. (실측: C3 1개=0.48, C3+C4=0.57, 64+CSP는 발 문제에서 0.71로 압승.)

## 2.3 목표 = 6개의 2지선다 게임, 평균 60%

각각은 **둘 중 하나 고르기**. 무엇과 무엇을 가르냐만 다름:

| 실험 | runs | 무엇 vs 무엇 |
|---|---|---|
| 0 | 3,7,11 | 실제 **왼손 vs 오른손** |
| 1 | 4,8,12 | 상상 **왼손 vs 오른손** |
| 2 | 5,9,13 | 실제 **양손 vs 양발** |
| 3 | 6,10,14 | 상상 **양손 vs 양발** |
| 4 | 3,7,11 vs 4,8,12 | **실제 vs 상상**(손) |
| 5 | 5,9,13 vs 6,10,14 | **실제 vs 상상**(양손/발) |

**목표**: 이 6문제를 109명 전부 풀고 → **6개의 실험 평균** → 그 평균이 **≥ 60%**. (우리 0.658.)

## 2.4 오려내기(에폭) = 45문제, "321"의 정체

- 데이터에 **동작 표시**가 미리 찍혀 있음: **T0=휴식, T1=한 동작, T2=다른 동작**.
- T0(휴식)은 버리고, **T1·T2 표시마다 그 지점부터 2초를 오려냄**(에폭). 한 조각 = **문제 한 장**.
- 한 조각 = **64채널 × 321칸**. **321 = 2초 × 160 + 1**.
  - *왜 +1?* 울타리 기둥 문제. 0초부터 2초까지 160번/초로 찍으면 **틈(간격)은 320개, 사진(샘플)은 321개**(시작점 0초의 첫 사진이 틈 없이 하나 더).
- run 하나에 T1+T2 약 15문제, 한 실험은 run 3개 → **한 사람·한 실험 ≈ 45문제**. 정답표 y는 거의 반반(예: 23 vs 22).
- (run 그룹마다 T1/T2의 *뜻*이 정해짐: run 3·7·11은 T1=왼손/T2=오른손, run 5·9·13은 T1=양손/T2=양발.)

## 2.5 ERD = "쓰면 반대쪽이 조용해진다" (이게 가능한 이유)

- 뇌의 한 구역이 **쉴 때**는 신경세포들이 **장단 맞춰 박수치듯** 8–30Hz로 크게 출렁임(큰 리듬).
- 그 구역을 **쓰면**(손 움직이거나 상상하면) 박자가 깨져 **출렁임이 작아짐** = **ERD**(Event-Related Desynchronization).
- 뇌는 반대쪽 담당이라: **오른손 → 왼쪽 뇌(C3) 조용**, **왼손 → 오른쪽 뇌(C4) 조용**.
- **실측(20명 평균, µV²)**: 왼손 C3=83.1 / C4=**78.7**(반대쪽 C4 조용), 오른손 C3=**82.2** / C4=89.3(반대쪽 C3 조용). 쏠림지수 C3/(C3+C4): 왼손 0.516 > 오른손 0.485. → **방향대로 맞음(작지만 진짜).**
- **핵심**: 답은 신호의 정확한 값이 아니라 **"출렁임의 크기(=분산)"**에 있다. 효과가 미묘해서 → 필터·CSP·LDA로 키운다.

## 2.6 필터 = ① 평균 기준 + ② 7–30Hz firwin

순서: **(주파수 무관) 64채널 평균 빼기 → 그 결과에서 7–30Hz만 추출.**

1. **평균 기준(`set_eeg_reference("average")`)**: 매 순간 64채널 평균을 빼서 **모두에 공통으로 깔린 잡신호 제거** → 각 채널 고유 활동만. (교실 전체 웅성거림 빼면 각자 말소리가 또렷.)
2. **7–30Hz 통과(firwin)**: ERD는 **뮤(8–12)+베타(13–30)**에만 삶. 그 아래(느린 드리프트)·위(근육·60Hz 전기잡음)는 버림.
   - **firwin** = FIR 필터를 '창 방법'으로 설계하는 레시피. 각 점을 **양옆 이웃의 가중평균**으로 다시 계산. **위상(타이밍)을 안 밀어** ERD 시점 보존, 절대 발산 안 함.
3. **순서 메모**: 긴 125초 신호에 **먼저 필터** 걸고 → 2초씩 오려냄(짧은 토막에 걸면 가장자리 망가짐).
- **실측(C3 주파수별 힘)**: 필터 전 0–7Hz=3743 / 7–30Hz=913 / 30–80Hz=162 → 필터 후 **8 / 164 / 7**. 즉 ERD(913)가 4배 큰 드리프트(3743)에 파묻혔다가, 필터 후 **ERD만 우뚝**.
- **뮤·베타**: 뇌파를 빠르기로 나눈 이름표. 뮤=운동구역의 "공회전 소리"(쉴 때 ~10Hz), 베타=운동 활동 리듬. 둘 다 움직이면 줄어듦(ERD).

## 2.7 CSP = 3걸음 (64×321 → 4숫자)

직접 구현 핵심. **"한 문제(2만 숫자)를, 두 동작이 잘 갈리는 4숫자로 압축."**

**① 왜 줄이나**
- 답은 ERD='출렁임 크기'에만 있으니 2만 숫자 다 필요 없음.
- 게다가 한 사람 45문제뿐인데 2만 차원에 울타리를 못 그음(과적합) → **4숫자로 줄여야** 깔끔한 울타리.

**② 어떻게 섞나 (레시피 + `@`)**
- **레시피(공간 필터)** = 채널마다 가중치 하나씩 **64개 숫자 묶음**. (스무디 배합표: FC4를 −8.5만큼, C4를 −5.9만큼...) 4개 보관(`filters_` = 4×64).
- **`@` = "곱해서 다 더하기"**: `레시피 @ E`는 64채널을 가중합해 **가상 채널(1줄)**을 만듦. 모든 시간칸에 동시에.
  - 작은 예: 가중치 `[2,0,-1]` · 값 `[10,5,4]` → 2·10+0·5−1·4 = 16.
- **축소는 두 번**: `64×321 ─@→ 4×321 ─각 줄 분산→ 4숫자`. ① 채널 64→4, ② **시간 321→1**(출렁임 크기만, 모양·타이밍 버림). 그래서 최종은 4×321이 아니라 **숫자 4개**.
- 실측: 레시피 ①이 크게 주목한 채널 = FC4·C4·C6·FC6·CP4 → **전부 오른쪽 운동구역**(= "오른쪽 운동구역 감지기"). CSP가 스스로 고름.

**③ 어떻게 레시피를 정하나 (공분산 → eigh → λ)**
- **클래스별 공분산 C1, C2**(64×64): 각 동작에서 "채널들이 어떻게 엮여 출렁이나" 패턴.
- **`eigh(C1, C1+C2)`**(일반화 고유분해): `wᵀC1w / wᵀ(C1+C2)w` 비율을 최대화하는 방향(레시피)들과 점수 λ를 한 번에.
- **λ = a/(a+b)** = 그 레시피 출렁임 중 한 클래스가 차지한 비율. λ→1(한쪽 전용), λ→0(다른쪽 전용), **λ=0.5(쓸모없음)**.
  - 실측: 레시피1 λ=0.378(a=0.378, b=0.622), 레시피2 λ=0.619 — λ과 a/(a+b)가 완전히 같음.
- **0.5에서 제일 먼 4개**를 골라 `filters_`에 보관.

**④ 특징 = 로그-분산 (transform)**
- 레시피로 섞어(@) 가상 채널 4줄 → 각 **분산**(출렁임 크기) → **비율로 정규화** → **log** → 4숫자.
- **왜 log?** ① 분산은 한쪽으로 쏠린 분포 → log가 종 모양(가우시안)으로 펴줌(LDA가 좋아함). ② "몇 배 차이(곱셈)"를 "얼마 차이(덧셈)"로 바꿔 직선 울타리가 가르기 쉬움(데시벨·리히터와 같은 이유). 실측: log 없이 0.884 < log 0.896.

> **용어 한 장**: 레시피 = 공간필터 = **고유벡터(eigenvector)** = `filters_` 한 줄 = 64가중치 / 점수 = **λ(람다)** = **고유값(eigenvalue)** = a/(a+b). **레시피 1개 ↔ λ 1개**.
> 이름 앞 `_`(`_class_cov`)=내부용, 뒤 `_`(`filters_`)=fit 후 학습으로 생긴 값(sklearn 규칙).

**PCA가 아니라 CSP인 이유**: PCA는 정답 안 보고(비지도) '가장 큰 변화'를 찾아 큰 잡음(눈 깜빡임)에 낚임. CSP는 정답 보고(지도) '두 클래스의 **차이**'를 직접 노림. 우리 일은 '구별'이라 CSP.

## 2.8 LDA = 두 무리에 울타리 긋기 (4숫자 → 판정)

- CSP의 4숫자 = **4차원 점**. 양손 점들·양발 점들이 각각 뭉침(두 무리).
- **선이 2개**(헷갈림 주의):
  - **방향 `w`**(화살표) = 두 무리를 **관통**(중심→중심). ← 우리가 계산하는 것.
  - **울타리** = 그 `w`에 **수직**으로 두 무리 **사이**를 가름. (자동으로 따라옴.)
- **수학**: μ0, μ1 = 두 무리 중심. **`w = Σ⁻¹(μ1−μ0)`** = 중심차를 **퍼짐(Σ)으로 보정**한 방향. (`Σ⁻¹`=나누기의 행렬판 → 많이 퍼진 노이즈 축은 덜 믿음.) `b` = 울타리 위치.
- **판정**: `decision = w·x + b`의 **부호**(양수→class1, 음수→class0). `w·x`=내적=곱해서 다 더하기.
- **LDA도 4→1 차원축소를 *함***: `w`를 자(ruler)로 보고 점을 떨어뜨린 값이 1숫자. 다만 거기서 멈추지 않고 **눈금 b와 비교해 클래스를 출력**해서 "분류기". (CSP=줄여서 재료, LDA=줄여서 결론.)
- **실측(exp2, 양손/양발)**: μ0=[-1.93,-1.78,-1.07,-1.13], μ1=[-1.11,-1.22,-1.79,-1.77], w=[12.92,7.86,-10.4,-2.13], b=13.46 → 양손 전부 음수, 양발 전부 양수 = **100% 분리**. (보정 효과: 중심차 3·4번 비슷했는데 w에선 −10.4 vs −2.13 — 깨끗한 3번은 크게 믿고 노이즈 큰 4번은 덜 믿음.)

## 2.9 교차검증 + 데이터 누수 + A/B (정직한 채점)

**왜 그냥 채점 안 되나**: 배운 문제로 시험 = 컨닝(외운 것). 실측: 같은 데이터로 학습·시험 = **1.0**(부풀려짐).

**교차검증(`cross_val_score`)**:
- 45문제를 **매 라운드 랜덤 20%(9개) 숨기고 80%(36개)로 학습 → 9개로 채점**, 이걸 **10번 반복**(`ShuffleSplit(10, test_size=0.2)`) → 점수 10개 평균.
- 한 번은 운에 휘둘림 → 10번 평균이라야 안정적. **fold = 한 라운드**.
- 실측(train 4 14): `[0.4444, 1, 0.5556, ...]`(각 k/9) → 평균 **0.7333**.
- `cross_val_score`는 그 for문의 포장지: 매 라운드 **새 빈 모델을 fit→채점→버림**. 한 번 쭉 학습이 아님.

**데이터 누수(leakage)**: CSP도 데이터에서 배움 → CSP가 시험문제까지 보면 컨닝.
- 막는 법: **CSP를 Pipeline 첫 단계로** 넣어, `cross_val_score`가 **fold마다 CSP+LDA를 학습분으로만 재학습**.
- 실측: CSP가 전부 엿본 누수=**1.0** vs Pipeline 안 재학습=**0.844** → **0.156 부풀림.**
- 규칙: *배우는* 단계(CSP·LDA)는 fold 학습분만. 안 배우는 필터·오려내기는 미리 한 번 OK.

**A/B 두 가지 (train이 둘 다 함)**:
| | 분할 | 데이터 | 목적 | 모델 |
|---|---|---|---|---|
| **A 채점** | ShuffleSplit 10번 | 45개 전부(랜덤 80/20) | **공식 점수**(0.7333)·60% 게이트 | 10개 다 버림 |
| **B 저장** | train_test_split 1번 | 고정 80%(36개) | **predict가 굴릴 모델** | .joblib 저장 |
- A·B는 독립. **A가 진짜 결과(점수)**, B는 predict용 부산물. A가 필요한 이유: 한 번 분할(B)의 점수는 운에 휘둘려 못 믿음 → 10번 평균(A).
- B의 남은 20%(9개)는 학습에 안 쓰고 저장 → **predict의 "처음 보는 시험지".**

## 2.10 sklearn 통합 = "표준 플러그"

- **sklearn** = 머신러닝 표준 부품 공장(분류기·도구·연결 규격 제공). **Pipeline** = 컨베이어 조립 라인(CSP칸 → LDA칸).
- **상속** = 부모 클래스 능력을 물려받기. `class MyCSP(TransformerMixin, BaseEstimator)` = 두 부모의 자격을 물려받음.
  - **BaseEstimator** → `get_params`/`set_params`("신분증" — sklearn이 **복제(clone)·튜닝** 가능).
  - **TransformerMixin** → `fit_transform`("변환기 자격").
- **왜 꼭 상속?** 우리가 CSP를 직접 만들었으니, sklearn이 "내 표준 부품"으로 인정하게 규격을 맞추는 것. 그래야 **Pipeline에 꽂히고**, **`cross_val_score`가 fold마다 clone·재학습**(=누수 방지)할 수 있음. 형식이 아니라 **정직한 채점의 토대**.

## 2.11 사람별 모델 (범용 아님)

- 모델은 **한 사람에 맵핑** = "누구냐"가 아니라 **"무슨 동작이냐"**를, 그 사람 전용으로 맞힘.
- EEG는 사람마다 뇌·전극이 달라 **사람별 모델이 표준**(다른 사람엔 캘리브레이션=새로 학습).
- CSP+LDA는 **수초·해석적**(고유분해 한 방 + 공식 한 방). 경사하강 장시간 도는 딥러닝과 다름.
- 60% 게이트도 **109명 각자에게 모델 하나씩 학습→채점→평균**.

## 🎬 Implementation (차원축소 직접 구현) — 배점 큰 핵심
**📋** 차원축소 알고리즘 구현 + **알고리즘 전반 이해**. (고유값분해·SVD·공분산에 numpy/scipy 허용.)

**🗣️ + 💻** "CSP를 밑바닥부터 직접 구현(`csp.py`). import는 `numpy`, `scipy.linalg.eigh`(허용), `sklearn.base`, 자작 `generalized_eigh`뿐 — **라이브러리 CSP 없음.**"

**🗣️ 알고리즘 4단계** (2.7 그대로): ① 왜 줄이나(답은 출렁임 크기, 45문제로 2만차원 못 그음) → ② 공분산 C1,C2 → ③ `eigh(C1,C1+C2)`로 쏠리는 레시피+λ(=a/(a+b)), 0.5에서 먼 4개 → ④ 섞어서(@) 로그-분산 4숫자.

**💻 검증**: `pytest tests/test_csp_parity.py -v` → **PASSED** (우리 CSP가 mne CSP와 같은 분할에서 <0.05 일치, ≥0.60).
> (단일 노이즈 피험자 정확도 비교는 시험 9개라 들쭉날쭉 — 공정한 비교는 안정적 exp3·동일 분할의 패리티 테스트.)

**🌟** "고유분해마저 직접(Jacobi) 구현한 보너스가 있는데 **보너스 항목에서 시연**."

**❓ (이해도)**
- *eigenvalue(λ)?* "그 레시피 출렁임 중 한 클래스 비율. 0.5에서 멀수록 잘 가름."
- *공분산은?* "각 클래스에서 채널들이 어떻게 같이 출렁이나(64×64). CSP 재료."
- *왜 CSP, PCA 아니고?* "PCA=비지도(큰 변화, 잡음에 낚임), CSP=지도(두 클래스 차이). 우리는 '구별'이라 CSP."
- *왜 로그-분산?* "쏠린 분포를 종 모양으로, 곱셈차를 덧셈차로 → 직선 분류기에 유리."
- *왜 일반화 고유분해(두 행렬)?* "비율 wᵀC1w/wᵀ(C1+C2)w 최대화라 분모에 (C1+C2)."

→ **Yes** ✅

**`csp.py` 블록별 (깊은 질문 대비)**:
- `__init__`(13–17): 하이퍼파라미터만 저장(계산 없음, sklearn 규약).
- `_class_cov`(19–35): `E@E.T`(한 조각 공분산) → `trace` 정규화(센 조각이 평균 지배 못하게) → 클래스 평균 → 대칭화(부동소수점 보정) → **shrinkage**(`reg=0.01`, 대각 섞어 특이행렬 방지 → eigh 안정).
- `fit`(37–66): `y` 필수(지도학습) → C1,C2 → `eigh(C1,C1+C2)` → `argsort(|λ-0.5|)[::-1]`(쏠린 순) → `filters_`(상위 4×64), `patterns_`(시각화용 유사역행렬).
- `transform`(68–78): `filters_@E`(가상채널) → 분산 → 정규화 → log → 4숫자.
- `filters_` vs `patterns_`: 전자=신호→특징(분석), 후자=특징→두피분포(시각화).
- `np.errstate(...)`: macOS BLAS 가짜 경고 억제(값은 정확).

## 🎬 Bonus · Feature engineering (보강)
**📋** 전처리 적절성 + 데이터 입력 방식. **푸리에·웨이블릿 등 처리 전 변환은 가산점.**

**🗣️ 두 층의 주파수 변환:**
> "① 필수 전처리부터 **FIR 밴드패스(7–30Hz)** — 이미 주파수 영역 변환입니다.
> ② 보너스 **Filter-Bank CSP(FBCSP)**: 7–30Hz를 **4개 하위 밴드(8-12 / 12-16 / 16-20 / 20-30)로 쪼개 밴드마다 따로 CSP** → 각 2성분 = **8숫자**."

**🗣️ *왜* 쪼개는 게 의미 있나 (깊이):**
> "단일 7–30Hz는 **뮤·저베타·고베타를 한 덩어리로 뭉뚱그립니다.** 그런데 **ERD의 세기와 두피 분포는 주파수마다 달라요** — 뮤에서 강한 사람, 베타에서 강한 사람이 다릅니다. FBCSP는 밴드를 쪼개 **밴드마다 다른 공간패턴(레시피)을 따로 학습**하고, 분류기가 **어느 밴드가 판별에 중요한지 가중치로 배웁니다.** 단일 CSP가 못 하는 '주파수 해상도'가 생기는 거죠. 이는 푸리에/웨이블릿과 같은 **주파수 영역 특징공학** 계열입니다."

**🗣️ 데이터가 어떻게 입력되나:**
> "각 하위 밴드 → 그 밴드용 CSP → 로그-분산 특징 → **이어붙여(concatenate)** 분류기에 한 번에 줍니다. 밴드패스를 `fit`/`transform` **안**에 둬서 cross_val_score가 **fold마다 재학습**(누수 없음)합니다."

**💻** `[F] Filter-Bank CSP (4 sub-bands) : 0.8222` (기본 CSP 0.8444)
**🗣️ 정직하게:** "이 피험자에선 0.822로 기본(0.844)보다 약간 낮습니다. 밴드를 쪼개면 **특징이 8개로 늘어 작은 데이터에서 약간 과적합**할 수 있어요. 하지만 FBCSP는 **엄밀히 더 표현력 큰 방법**이고, 다른 피험자·데이터셋에선 이길 수 있습니다. **더 정교한 특징공학을 구현·시연**하는 의미입니다."

**🗣️ 코드(`fbcsp.py`):** `DEFAULT_BANDS` 4개 → `fit`이 밴드마다 `MyCSP`를 학습(`csps_`) → `transform`이 밴드별 특징을 이어붙임. 밴드 필터(`mne.filter.filter_data`)는 fit/transform 내부.

**❓**
- *FBCSP가 단일 CSP보다 더 하는 일?* "주파수를 하위밴드로 나눠 밴드별 공간패턴을 따로 뽑아, 분류기가 밴드 중요도를 학습."
- *왜 더 정확하지 않나?* "이 데이터·피험자에선 특징 증가로 약간 과적합. 항상 우월하진 않은, 더 정교한 기법 시연."
- *웨이블릿/푸리에를 직접?* "필터뱅크가 그 계열의 변환입니다 — 주파수를 밴드로 분해해 특징을 만듭니다."

→ **Yes** ✅

## 🎬 Bonus · Implementations (보강)
**📋** 얼마나 깊이 — **자작 고유분해/SVD/공분산? 복잡한 차원축소? 튜닝/학습? 자작 분류기?**

### A. 자작 고유분해 (Jacobi) — "own eigendecomposition"
**🗣️** "`eigh`마저 라이브러리 없이 **순수 numpy로 cyclic Jacobi** 알고리즘을 짰습니다(`jacobi.py`)."
**🗣️ *어떻게* 도나 (깊이):** "행렬의 **비대각 원소를 한 쌍씩 회전으로 0**으로 만들길 반복(sweep)합니다. 비대각이 충분히 작아지면 멈추고 — **남은 대각이 고유값**, 누적한 회전이 **고유벡터**입니다. 일반화 고유분해 `C1 w = λ(C1+C2)w`는 **백색화(whitening)**로 표준 문제로 바꾼 뒤 Jacobi를 두 번 적용합니다. 쓰는 연산은 `+,*,sqrt,sign,matmul`뿐 — `np.linalg.eig/eigh/svd`나 scipy를 안 부릅니다."
**💻**
```
[A] Jacobi 정확도            : 0.8444  (scipy eigh와 동일)
    표준 고유분해 vs numpy   : 5.33e-14   ← 기계 정밀도
    일반화 고유분해 vs scipy : 1.55e-15   ← 사실상 완벽
```
**🗣️** "고유분해는 보통 블랙박스 라이브러리 호출인데, **밑바닥부터 짜서도 5e-14로 일치** — 핵심 선형대수를 이해했다는 증거입니다." `MyCSP(solver="jacobi")`로 켭니다.

### C. 하이퍼파라미터 튜닝 — "tuning/learning"
**🗣️** "`GridSearchCV`로 `n_components ∈ {4,6,8}`을 탐색합니다(`evaluate.tune`). **inner fold마다 CSP를 재학습**해 **누수 없는 중첩 교차검증**입니다."
**💻** `[C] tuned {'csp__n_components': 8} -> cv 0.8444`
**🗣️ 깊이:** "이건 **모델 선택(model selection)** + **중첩 CV의 누수 방지**를 이해했다는 부분입니다. 단, run_all(공식 게이트)은 비교 공정성을 위해 **고정 4성분**을 유지하고, 튜닝은 데모로만 둡니다."

### D. 자작 분류기 (OwnLDA) — "own classifier"
**🗣️** "LDA도 직접 구현(`own_lda.py`). 풀드 공분산 → **`w = Σ⁻¹(μ1−μ0)`**, 절편은 사전확률 보정, 판정은 `w·x+b` 부호. `np.linalg.solve`만 씁니다."
**💻**
```
[D] OwnLDA 정확도          : 0.7889
    sklearn LDA 예측 일치율 : 100.0%   ← 완전 동일
```
**🗣️ 깊이:** "분류기를 `LDA()` 호출이 아니라 **수식으로 직접** 구현해 sklearn과 100% 일치 — 분류기 수학을 이해했다는 증거입니다." `build_pipeline(clf="own-lda")`로 켭니다.

### (+ F. FBCSP = "복잡한 차원축소"에도 해당)

**❓**
- *Jacobi가 진짜 라이브러리 없이?* "`+,*,sqrt,sign,matmul`만. eig/eigh/svd 안 부름. 그래도 scipy와 5e-14."
- *일반화 고유분해 방법?* "백색화로 표준 문제화 후 Jacobi 두 번."
- *튜닝 누수 없나?* "inner fold 학습분으로만 CSP 재학습."
- *OwnLDA가 sklearn과 같나?* "예측 100% 일치(같은 풀드 공분산 + Σ⁻¹(μ1−μ0))."

→ **Yes** ✅
