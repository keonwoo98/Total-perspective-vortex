# 2부. 핵심 개념 완전 복습 (라이브로 설명한 깊이 그대로)

## 2.1 EEG = 머리에 붙인 마이크 64개

머리에 전극(=작은 마이크)을 **64개** 붙이고, 각 전극이 **초당 160번** 그 자리의 전압을 잽니다. 단위는 **마이크로볼트(µV)**.

- **마이크 하나 = 숫자 한 줄.** 시간 순서대로 찍힌 숫자들을 점으로 이으면 **출렁이는 선**이 됩니다.
- **64개가 동시에** 이어지니 → **64줄짜리 표**. 가로 = 시간, 세로 = 64채널.
- 한 사람의 한 녹음(run)은 약 125초. **125초 × 160번/초 = 20000칸** → 표 모양 **(64, 20000)**.

**실제 표 한 귀퉁이 (피험자1, µV):**
```
        시간1   시간2   시간3   시간4   시간5
 C3  |  -12.0  -19.0  -30.0  -53.0  -65.0
 Cz  |  -29.0  -41.0  -53.0  -65.0  -73.0
 C4  |  -15.0  -34.0  -48.0  -58.0  -64.0   ... (64채널 × 20000칸)
```
→ 칸 안의 숫자 = 그 순간 그 마이크의 µV. **여기까진 새로운 게 하나도 없음.**

## 2.2 채널 이름 = 머리 위치, 그리고 C3/C4

전극 이름은 아무거나가 아니라 **머리 어디에 붙었는지를 나타내는 좌표**입니다.

- **앞글자 = 구역**: `F`=이마(Frontal), **`C`=정수리·운동 구역(Central)**, `P`=뒤(Parietal), `O`=뒤통수(Occipital), `T`=옆/관자(Temporal).
- **뒷숫자 = 좌/우**: **홀수=왼쪽, 짝수=오른쪽, `z`=정중앙**.

그래서:
- **C3** = Central + 3(홀수) = 정수리 왼쪽 = 왼쪽 운동구역 → **오른손 담당** (표에서 9번째 채널).
- **C4** = Central + 4(짝수) = 정수리 오른쪽 = 오른쪽 운동구역 → **왼손 담당** (13번째).
- **Cz** = 한가운데 (11번째, 발 담당).
- **뇌는 반대쪽을 제어**합니다(좌우 교차). 이건 그냥 받아들이는 생물학적 사실.

64채널 중 **운동구역(C로 시작: C5 C3 C1 Cz C2 C4 C6, 7개)**이 손/발 동작의 핵심 자리. 나머지 57개(이마·옆·뒤통수)도 표엔 다 있지만 손 동작 신호는 상대적으로 약합니다.

**❓ "그럼 C3·C4 2개만 쓰면 되잖아?" (자주 나오는 질문 — 직접 실험해 답):**
- 손 좌우(exp0)에서는 실제로 2개로도 거의 됩니다. **10명 평균 실측**: C3 1개=**0.479**, C3+C4=**0.570**, 64+CSP=**0.568** — 2개와 비슷.
- 하지만 **양손 vs 양발(exp2)**: C3+C4 2개=**0.564** vs 64+CSP=**0.713**. 발은 정답이 C3/C4가 아니라 **Cz(정중앙)** 쪽에 있어서, 손에 맞춘 2채널로는 망합니다.
- → 우리는 문제가 **6종**이고 **문제마다 정답이 숨은 채널이 다름.** 그래서 채널을 손으로 못 박고, **64개를 다 주고 CSP가 "이 문제에선 어디가 잘 갈리나"를 자동으로** 찾게 합니다. (여러 채널을 섞으면 잡음도 상쇄돼 더 안정적.)

## 2.3 목표 = 6개의 "2지선다 게임", 평균 60%

이 프로젝트는 한 마디로 **맞히기 게임**입니다: *"이 표 토막을 보여줄게 — 무슨 동작이게?"* 6종인데 **각각은 여전히 둘 중 하나 고르기**이고, 무엇과 무엇을 가르냐만 다릅니다:

| 실험 | runs | 무엇 vs 무엇 (T1 vs T2) |
|---|---|---|
| 0 | 3,7,11 | 실제 **왼손 vs 오른손** |
| 1 | 4,8,12 | 상상 **왼손 vs 오른손** |
| 2 | 5,9,13 | 실제 **양손 vs 양발** |
| 3 | 6,10,14 | 상상 **양손 vs 양발** |
| 4 | 3,7,11 vs 4,8,12 | **실제 vs 상상**(손 과제) |
| 5 | 5,9,13 vs 6,10,14 | **실제 vs 상상**(양손/발 과제) |

연습 문제엔 **정답이 같이 붙어 있음**(단어카드 앞=표, 뒤=정답). 컴퓨터는 정답 달린 문제 수백 개로 연습하고 → 정답 안 보여준 새 문제를 맞힙니다.

**목표**: 이 6문제를 **109명 전부** 풀고 → **실험 유형별로 109명 평균(6개)** → 그 **6개의 평균이 ≥ 60%**. (우리 **0.658**. `python mybci.py` 하나가 이 6×109 평균을 계산.)

## 2.4 통짜 표를 낱개 문제로: 오려내기(에폭)와 "321"의 정체

125초짜리 통짜 표를 어떻게 문제 한 장씩으로 쪼개나?

**① 데이터에 "동작 표시"가 미리 찍혀 있다.** 시간 순서로 표시(annotation)가 박혀 있음:
```
 0.0초  T0  (쉬는 중)
 4.2초  T2  (한 동작 했음)
 8.3초  T0  (쉬는 중)
12.5초  T1  (다른 동작 했음)  ...
```
- **T0 = 휴식, T1 = 한 동작, T2 = 다른 동작.** 데이터셋이 미리 박아둔 정답입니다(우리가 만든 게 아님).
- run 그룹마다 T1/T2의 *뜻*이 정해져 있음: **run 3·7·11 → T1=왼손/T2=오른손**, **run 5·9·13 → T1=양손/T2=양발.**

**② 표시마다 2초를 오려낸다.** 휴식(T0)은 버리고, T1·T2 지점부터 **2초씩 오려냄**(= 에폭). 오려낸 조각 하나 = **문제 한 장**.
- 조각 모양 = **64채널 × 321칸.**

**③ "321"은 왜 320이 아니라 321인가? — 울타리 기둥 문제.**
작게 보면: **1초 동안 0.5초 간격으로 사진**을 찍으면 → 0초, 0.5초, 1.0초 = **사진 3장**인데, 그 사이 **틈(간격)은 2개**뿐.
```
사진:  ●——————●——————●
시각:  0초    0.5초   1.0초
틈:       1번     2번        ← 틈은 2개, 사진은 3장
```
→ **사진 수 = 틈 수 + 1** (시작점 0초의 첫 사진이 틈 없이 하나 더 있으니까).
우리 경우: 0~2.0초를 160번/초로 → **틈 = 2×160 = 320**, **사진(샘플) = 320 + 1 = 321**. 양 끝(0초·2.0초)을 둘 다 포함해서 딱 +1.

**④ 한 사람·한 실험 ≈ 45문제.** run 하나에 T1+T2 약 15문제, 실험은 run 3개 → **약 45문제**.
- 실측(exp0): run3=15, run7=15, run11=15 → **합 45**. 정답표 y는 거의 반반(왼손 23 / 오른손 22).
- 주의: "왼손 3런 + 오른손 3런"이 아니라 **한 실험에 3런, 각 런 안에 왼손·오른손이 섞여 15개씩.**

## 2.5 ERD = "쓰면 반대쪽이 조용해진다" (이게 가능한 *물리적* 이유)

**경기장 박수 비유:**
- 뇌의 한 구역이 **쉴 때**는 그 동네 신경세포들이 **다같이 장단 맞춰 박수치듯** 8–30Hz로 똑같이 깜빡임 → 합쳐지니 **출렁임이 큼**(큰 리듬).
- 그 구역을 **쓰기 시작하면**(손 움직이거나 상상) 세포들이 각자 바빠져 **박자가 깨짐** → 한 덩어리 리듬이 흩어져 **출렁임이 작아짐.**
- 이 "박자 깨짐"이 **D**e-synchronization. **ERD = Event-Related Desynchronization = "동작에 맞춰 그 구역 출렁임이 줄어드는 현상."**
- 뇌는 반대쪽 담당 → **오른손 → 왼쪽 뇌(C3) 박자 깨짐 → C3 출렁임↓**, 왼손 → C4 출렁임↓.

**실측(20명 평균, 출렁임 크기 µV²):**
```
         C3 출렁임   C4 출렁임
 왼손   :   83.1   >   78.7      → 반대쪽 C4가 더 조용 ✓
 오른손 :   82.2   <   89.3      → 반대쪽 C3가 더 조용 ✓
쏠림지수 C3/(C3+C4):  왼손 0.516 > 오른손 0.485 ✓
```
→ 교차(반대쪽) 패턴이 **방향대로 진짜로 있음.** 단 차이가 **미묘함**(80 대 89). 그래서 C3 하나만 보면 못 맞힘 → **필터·CSP·LDA로 이 미묘한 차이를 키운다.**

**핵심 메모**: 답은 신호의 정확한 값이 아니라 **"출렁임의 크기"**(= 분산, variance)에 있음. ERD는 모양·타이밍이 아니라 **크기**를 바꾸니까. → CSP가 분산을 보는 이유.

## 2.6 필터 = ① 평균 기준 + ② 7–30Hz firwin

코드는 두 줄: `set_eeg_reference("average")` + `filter(7, 30, "firwin")`. 순서는 **(주파수 무관) 평균 빼기 → 그 결과에서 7–30Hz 추출.**

**① 평균 기준 (average reference):**
EEG 전압은 항상 **"어떤 기준점 대비"** 값(절대값 아님). 기준점 때문에 **모든 채널에 똑같이 얹히는 공통 잡신호**가 생김.
- 매 순간 **64채널 평균을 빼면** → 공통으로 깔린 건 사라지고 **각 채널 고유 활동**만 남음.
- 비유: **교실 전체가 웅성대는 소음(공통)을 빼면 각자 말소리(고유)가 또렷.**

**② 7–30Hz만 통과 (band-pass):**
- ERD는 **뮤(8–12Hz) + 베타(13–30Hz)**에만 삶.
  - **뮤** = 운동구역의 "공회전 소리"(쉴 때 ~10Hz로 웅~). 움직이면 꺼짐 = ERD.
  - **베타** = 운동 활동/유지 리듬. 역시 움직이면 줄어듦.
- 그 아래(0–7Hz) = 땀·전극 움직임 같은 **느린 드리프트**. 위(30Hz~) = **근육 떨림·60Hz 전기 잡음.** 둘 다 답과 무관 → 버림. (8이 아닌 7로 살짝 여유.)

**③ firwin = 필터를 *어떻게* 만드냐의 레시피:**
- 필터는 각 시점 값을 **"양옆 이웃 값들의 가중평균"**으로 다시 계산. 빠른 출렁임은 이웃끼리 상쇄돼 죽고 적당한 리듬만 남음. firwin = 그 **가중치를 정하는 표준 레시피**(FIR 필터를 '창 방법'으로 설계).
- 쓰는 이유: **위상(타이밍)을 안 밀어** ERD 시점이 안 틀어짐(zero-phase), 절대 발산 안 함.
- 단, 7Hz·30Hz 경계는 **칼같은 수직 절벽이 아니라 가파른 비탈**(유한 FIR이라). 6.5나 31이 0이 되진 않고 점점 줄어듦.

**④ 순서 메모:** 코드는 **125초 통짜 신호에 필터 먼저** 걸고 → 2초씩 오려냄. 필터는 각 점 계산에 양옆 이웃을 많이 봐야 깨끗한데, 2초 토막에 걸면 가장자리가 망가지기 때문.

**실측(C3 주파수 구간별 힘):**
```
              0–7Hz(느린흔들림)  7–30Hz(ERD)  30–80Hz(잡음)
 필터 전          3743    >>>       913    >>     162
 필터 후             8               164            7
```
→ 필터 전엔 우리가 원하는 ERD(913)가 **4배 큰 드리프트(3743)에 파묻혀** 있었음. 필터 후엔 **ERD만 우뚝** 남음. 즉 필터의 진짜 일 = "잡소리를 걷어내 ERD를 드러내기."

## 2.7 CSP = 3걸음 (64×321 → 4숫자) — 직접 구현 핵심

한 마디: **"한 문제(2만 숫자)를, 두 동작이 잘 갈리는 4숫자로 압축."**

### ① 왜 줄이나
- **줄여도 된다**: 답은 ERD='출렁임 크기'에만 있으니 2만 숫자 다 필요 없음. 잘 고른 '출렁임 크기' 몇 개면 충분.
- **줄여야만 한다**: 한 사람 45문제뿐인데 숫자가 2만개면 울타리를 못 그음(그냥 외워버려 새 문제에 실패=과적합). **4개로 줄여야** 45문제로도 깔끔한 울타리.

### ② 어떻게 섞나 — 레시피 + `@`
- **레시피(=공간 필터)** = 채널마다 가중치 하나씩 **64개 숫자 묶음**. **스무디 배합표** 비유: "이 스무디 = FC4를 −8.5만큼, C4를 −5.9만큼, C6를 +3.7만큼… (64가지 재료)". 4개 보관(`filters_` = 4×64).
- **`@` = "곱해서 다 더하기"**. 작은 예:
  ```
  가중치 [2, 0, -1] 를 어떤 순간 채널값 [10, 5, 4] 에 →  2·10 + 0·5 + (-1)·4 = 16
  ```
  `레시피 @ E`는 이 "곱해서 더하기"를 **모든 시간 칸에** 해서 → **가상 채널(스무디 한 잔, 1줄)**을 만듦.
- 실측: 레시피 ①이 크게 주목한 채널 = FC4·C4·C6·FC6·CP4 → **전부 오른쪽 운동구역**(= "오른쪽 운동구역 감지기"). CSP가 스스로 고름. 그 가상 채널은 C4 원본보다 출렁임이 **더 뚜렷**(잡음 상쇄, 패턴 증폭).
- **축소는 두 번**:
  ```
  64×321 (2만) ─@→ 4×321 ─각 줄의 분산→ 4숫자
  ```
  ① 채널 64→4(섞기), ② **시간 321→1**(각 가상채널의 출렁임 크기 하나로, 모양·타이밍은 버림). 그래서 최종은 4×321이 아니라 **숫자 4개.**

### ③ 어떻게 레시피를 정하나 — 공분산 → eigh → λ
- **클래스별 공분산 C1, C2 (64×64)**: 각 동작에서 "어느 채널이 얼마나, 채널끼리 어떻게 같이 출렁이나"를 그 클래스 45문제 평균으로 담은 표. CSP의 재료.
- **`eigh(C1, C1+C2)` (일반화 고유분해)**: `wᵀC1w / wᵀ(C1+C2)w` 비율을 **최대로 만드는 방향(레시피)**들과 그 점수 **λ**를 한 방에 줌.
  - 이게 일반화인 이유: 보통 `A w = λ w`인데 여기선 분모에 (C1+C2)가 들어가 `C1 w = λ(C1+C2)w`, **두 행렬을 동시에** 봄.
- **λ = a/(a+b)** = 레시피 출렁임 중 한 클래스(a=왼쪽)가 차지한 **비율**:
  ```
  레시피  λ      왼쪽 a   오른쪽 b   a/(a+b)
    1    0.378   0.378    0.622     0.378   ← 오른쪽으로 쏠림(탐지기)
    2    0.619   0.619    0.381     0.619   ← 왼쪽으로 쏠림(탐지기)
  ```
  λ→1(한쪽 전용), λ→0(다른쪽 전용), **λ=0.5(양쪽 똑같음=쓸모없음)**. 그냥 합이 아니라 **비율**인 이유: 양쪽 다 크게 출렁이면 소용없으니, **상대적으로** 쏠려야 함.
- **0.5에서 제일 먼 4개**를 골라 `filters_`에 보관 (`argsort(|λ-0.5|)[::-1]`).

### ④ 특징 = 로그-분산 (transform)
- 레시피로 섞어(@) 가상채널 4줄 → 각 **분산**(출렁임 크기) → **비율로 정규화**(합=1, 조각의 전체 세기 차이 제거) → **log** → 4숫자.
- **왜 log? (두 가지)**
  1. 분산은 양수·**한쪽으로 쏠린 분포** → log가 펴서 **종 모양(가우시안)**으로 → 직선 분류기(LDA)가 좋아함.
  2. 출렁임은 "몇 배" 차이(곱셈)인데 log가 **"얼마 차이"(덧셈)**로 바꿔 → 직선 울타리가 가르기 쉬움. (데시벨·리히터 척도가 log인 것과 같은 이유.)
  - 실측: log 없이 0.884 < log 0.896.

> **용어 한 장 (헷갈림 방지)**: 한쪽은 "방향", 다른쪽은 "그 방향의 숫자":
> | 방향 | 그 방향의 점수 |
> |---|---|
> | **레시피** = 공간필터 = **고유벡터(eigenvector)** = `filters_` 한 줄 = 64가중치 | **λ(람다)** = **고유값(eigenvalue)** = `eigvals` 한 값 = a/(a+b) |
> 레시피 1개 ↔ λ 1개. / 이름 앞 `_`(`_class_cov`)=내부용, 뒤 `_`(`filters_`)=fit 후 학습으로 생긴 값(sklearn 규칙).

**PCA가 아니라 CSP인 이유**: PCA는 정답 안 보고(비지도) '가장 큰 변화'를 찾아 큰 잡음(눈 깜빡임)에 낚임. CSP는 정답 보고(지도) '두 클래스의 **차이**'를 직접 노림. 우리 일은 '구별'이라 CSP.

## 2.8 LDA = 두 무리에 울타리 긋기 (4숫자 → 판정)

CSP의 4숫자 = **4차원 점**. 양손 점들·양발 점들이 각각 뭉침(두 무리). LDA가 그 사이에 선을 긋습니다.

**선이 2개 (헷갈림 주의):**
- **방향 `w`(화살표)** = 두 무리를 **관통**(중심→중심). ← 우리가 *계산하는* 것.
- **울타리** = 그 `w`에 **수직**으로 두 무리 **사이**를 가름. (w가 정해지면 자동으로 따라옴.)
- 즉 `w`(관통)와 울타리(분리)는 **서로 직각.**

**자(ruler) 비유 — LDA도 차원축소를 *함*:**
- `w`를 **자**라고 보고, 점(4숫자)을 그 자에 **수직으로 떨어뜨림** → 자 위의 위치 숫자 하나 = `w·x`. 즉 **4숫자 → 1숫자.**
- 그 다음 자 위의 **눈금(b)**보다 작으면 class0, 크면 class1. 여기서 멈추지 않고 **클래스를 출력**해서 "분류기". (CSP=줄여서 재료, LDA=줄여서 결론.)

**수학:**
- `μ0, μ1` = 두 무리 중심(클래스별 4숫자 평균).
- **`w = Σ⁻¹(μ1 − μ0)`** = 중심→중심 방향(`μ1−μ0`)을 **무리 퍼짐 `Σ`로 보정**. `Σ⁻¹`(역행렬)="나누기의 행렬판" → 많이 퍼진(노이즈 큰) 축은 나눠서 **덜 믿음**.
- `b` = 울타리 위치. 판정: **`w·x + b`의 부호**(양수→class1, 음수→class0). `w·x`=내적=곱해서 다 더하기.

**실측(exp2, 양손 vs 양발):**
```
μ0(양손) [-1.93,-1.78,-1.07,-1.13]   μ1(양발) [-1.11,-1.22,-1.79,-1.77]
중심차 μ1-μ0 = [0.82, 0.56, -0.71, -0.65]
보정한 w     = [12.92, 7.86, -10.4, -2.13]   b = 13.46
→ 양손 전부 음수, 양발 전부 양수 = 100% 분리
```
보정 효과: 중심차 3번·4번은 −0.71/−0.65로 비슷했는데 w에선 **−10.4 vs −2.13** — 3번 축은 깨끗히 다르니 크게 믿고, 4번 축은 노이즈 커서 덜 믿음(`Σ⁻¹`이 한 일).

## 2.9 교차검증 + 데이터 누수 + A/B (정직한 채점)

**왜 그냥 채점하면 안 되나 — 컨닝:**
- 배운 문제로 시험 보면 외운 걸 되뱉어 점수가 부풀려짐. 실측: **같은 데이터로 학습·시험 = 1.0**(의심스러운 만점).
- 비유: 모의고사 문제+정답을 통째로 외워서 똑같은 문제로 시험 본 것.

**교차검증 (`cross_val_score`):**
- 45문제를 **매 라운드 랜덤 20%(9개) 숨기고 80%(36개)로 학습 → 숨긴 9개로 채점**, 이걸 **10번 반복**(`ShuffleSplit(10, test_size=0.2)`) → 점수 10개 평균.
- 한 번은 운에 휘둘림(숨긴 9개가 쉬웠/어려웠을 수도) → **10번 평균**이라야 안정적. **fold = 한 라운드.**
- 숫자 출처: 45 = 문제 수, 9 = 45의 20%, 36 = 나머지 80%, 10 = `n_splits`(우리가 정한 반복 횟수).
- 실측(train 4 14): `[0.4444, 1, 0.5556, 0.6667, ...]` → 평균 **0.7333**. 각 값이 **k/9**(0.4444=4/9, 9개만 시험하니).

**`cross_val_score`는 마법이 아니라 그 for문의 포장지:**
```
매 라운드:  새 빈 모델 만들기 → 36개로 학습 → 숨긴 9개로 채점 → 모델 버림
            (이걸 10번, 매번 다른 랜덤 9개로)
```
손으로 짠 루프와 `cross_val_score`가 **완전히 같은 배열**을 냄. 즉 **한 번 쭉 학습하고 마지막에 시험이 아니라, 처음부터 새로 학습+채점을 10번 반복.**

**데이터 누수(leakage) — CSP를 통한 미묘한 컨닝:**
- CSP도 데이터에서 배움(45문제 보고 레시피 계산). **CSP가 레시피 만들 때 숨길 시험문제까지 보면** → 이미 엿본 컨닝.
- 막는 법: **CSP를 Pipeline 첫 단계로** 넣어, `cross_val_score`가 **fold마다 CSP+LDA를 그 fold 학습분으로만 다시 학습.** (증거: fold마다 `filters_`가 다름.)
- 실측: CSP가 전부 엿본 누수=**1.0** vs Pipeline 안 fold별 재학습=**0.844** → **0.156 부풀림.**
- 규칙: *배우는* 단계(CSP·LDA)는 fold 학습분만 봐야 함. 안 배우는 필터·오려내기는 미리 한 번 OK.

**A/B 두 가지 (train이 둘 다 함):**
| | 분할 | 데이터 | 목적 | 모델 |
|---|---|---|---|---|
| **A 채점** | `ShuffleSplit(10, 0.2)` | 45개 전부(랜덤 80/20, 10번) | **공식 점수**(0.7333)·60% 게이트 | 10개 다 버림 |
| **B 저장** | `train_test_split(0.2)` 1번 | 고정 80%(36개) | **predict가 굴릴 모델** | `.joblib` 저장 |
- A·B는 **독립**. A가 진짜 결과(점수), B는 predict용 부산물.
- **왜 A가 필요?** B 한 번 분할의 점수는 운에 휘둘려 못 믿음(fold가 0.44~1.0로 출렁) → 10번 평균(A)이라야 정직. 그리고 `cross_val_score`는 모델을 안 남기니, 굴릴 모델은 B로 따로 만들어야 함.
- 저장된 `.joblib` 내용(실측): `pipeline`(80% 학습), `X_test (9,64,321)`(학습에 안 쓴 20%), `y_test`, `meta`. → predict가 그 9개를 "처음 보는 시험지"로 재생.
- 그래서 **predict는 train한 그 피험자·run으로만** 가능(파일명이 `subj004_run14_exp3.joblib`). 다른 사람은 새로 train.

## 2.10 sklearn 통합 = "표준 플러그"

- **sklearn** = 머신러닝 표준 부품 공장(분류기 LDA·도구 cross_val_score/GridSearchCV·연결 규격 제공).
- **Pipeline** = 컨베이어 조립 라인: `한 문제 → [CSP 칸: 2만→4] → [LDA 칸: 4→판정] → 결과.`

**상속(inheritance) = 물려받기 (장난감 코드):**
```python
class 부모:
    def 인사(self): return "부모한테 물려받은 기능"
class 자식(부모):      # 괄호 안에 부모 = 상속
    pass               # 자식엔 아무것도 안 적었는데
자식().인사()          # → "부모한테 물려받은 기능"  (그대로 씀!)
```
→ 우리 `class MyCSP(TransformerMixin, BaseEstimator)`도 똑같이 두 부모의 능력을 물려받음. (그래서 `fit_transform`을 **안 적었는데도** 있음 — TransformerMixin한테 물려받아서.)

**두 부모가 주는 것:**
| 상속 | 물려주는 것 | 한 마디 |
|---|---|---|
| **BaseEstimator** | `get_params`/`set_params` | **신분증** — sklearn이 **복제(clone)·튜닝·검사** 가능 |
| **TransformerMixin** | `fit_transform` 공짜 | **변환기 자격** — "데이터를 변환하는 부품" |

**왜 꼭 상속? — 콘센트와 플러그:**
- sklearn 도구(Pipeline, cross_val_score)는 **표준 콘센트**. 우리 CSP가 아무 모양이면 안 꽂힘. 두 클래스를 상속하면 **표준 플러그**가 되어 꽂힘.
- **결정적 이유**: `cross_val_score`가 누수 없이 채점하려면 **fold마다 CSP를 clone해서 다시 fit** 해야 하는데, 그 `clone`은 **BaseEstimator의 get/set_params가 있어야** 작동. 즉 상속을 안 했으면 **누수 없는 정직한 채점 자체가 불가능.** 형식이 아니라 토대.
- 한 줄: **직접 만든 CSP를 sklearn 표준 부품으로 인정받게 하려고 상속 → Pipeline에 꽂히고 + cross_val_score가 fold마다 복제·재학습.**

## 2.11 사람별 모델 (범용 아님)

- 모델은 **한 사람에 맵핑됨.** "이게 109명 중 누구냐"를 고르는 게 아니라, **그 한 사람의 "무슨 동작이냐"**를 맞힘.
- 다른 사람 뇌파를 그 모델에 넣으면 돌아가긴 하나 **정확도 급락**(레시피가 그 사람 뇌에 맞춰져 있어서). 새 사람은 **그 사람 데이터로 새로 학습**(=캘리브레이션, 몇 초)하면 됨. → 109명에 갇힌 게 아님.
- EEG는 사람마다 뇌·전극이 달라 **사람별 모델이 BCI 표준**(결함 아님, 오히려 더 잘 맞음).
- 학습 방식: CSP(고유분해 한 방) + LDA(공식 한 방) = **수초·해석적.** 경사하강법으로 오래 도는 **딥러닝(MLP) 범용 모델과 다름.**
  | | 우리(고전 BCI) | 딥러닝 범용 |
  |---|---|---|
  | 범위 | 사람별 전용(캘리브레이션) | 여러 사람 통합 |
  | 학습 | 수초·해석적 | 장시간·반복 |
  | 데이터 | 한 사람 수십 문제 | 대량·다수 |
- 60% 게이트도 **109명 각자에게 모델 하나씩 학습→채점→평균** (하나의 범용 모델을 109명에 적용하는 게 아님).

**💻 검증**: `pytest tests/test_csp_parity.py -v` → **PASSED** (우리 CSP가 mne CSP와 같은 분할에서 <0.05 일치, ≥0.60).
> (단일 노이즈 피험자 정확도 비교는 시험 9개라 들쭉날쭉 — 공정한 비교는 안정적 exp3·동일 분할의 패리티 테스트.)

**🌟** "고유분해마저 직접(Jacobi) 구현한 보너스가 있는데 **보너스 항목에서 시연**."

**❓ (이해도)**
- *eigenvalue(λ)?* "그 레시피 출렁임 중 한 클래스 비율. 0.5에서 멀수록 잘 가름."
- *공분산은?* "각 클래스에서 채널들이 어떻게 같이 출렁이나(64×64). CSP 재료."
- *왜 CSP, PCA 아니고?* "PCA=비지도(큰 변화, 잡음에 낚임), CSP=지도(두 클래스 차이). 우리는 '구별'이라 CSP."
- *왜 로그-분산?* "쏠린 분포를 종 모양으로, 곱셈차를 덧셈차로 → 직선 분류기에 유리."
- *왜 일반화 고유분해(두 행렬)?* "비율 wᵀC1w/wᵀ(C1+C2)w 최대화라 분모에 (C1+C2)."

**`csp.py` 블록별 (깊은 질문 대비)**:
- `__init__`(13–17): 하이퍼파라미터만 저장(계산 없음, sklearn 규약).
- `_class_cov`(19–35): `E@E.T`(한 조각 공분산) → `trace` 정규화(센 조각이 평균 지배 못하게) → 클래스 평균 → 대칭화(부동소수점 보정) → **shrinkage**(`reg=0.01`, 대각 섞어 특이행렬 방지 → eigh 안정).
- `fit`(37–66): `y` 필수(지도학습) → C1,C2 → `eigh(C1,C1+C2)` → `argsort(|λ-0.5|)[::-1]`(쏠린 순) → `filters_`(상위 4×64), `patterns_`(시각화용 유사역행렬).
- `transform`(68–78): `filters_@E`(가상채널) → 분산 → 정규화 → log → 4숫자.
- `filters_` vs `patterns_`: 전자=신호→특징(분석), 후자=특징→두피분포(시각화).
- `np.errstate(...)`: macOS BLAS 가짜 경고 억제(값은 정확).

→ **Yes** ✅

**🗣️ 데이터가 어떻게 입력되나:**
> "각 하위 밴드 → 그 밴드용 CSP → 로그-분산 특징 → **이어붙여(concatenate)** 분류기에 한 번에. 밴드패스를 `fit`/`transform` **안**에 둬 cross_val_score가 **fold마다 재학습**(누수 없음)."

**🗣️ 정직하게:** "이 피험자에선 0.822로 기본(0.844)보다 약간 낮습니다. 밴드를 쪼개면 **특징이 8개로 늘어 작은 데이터에서 약간 과적합**할 수 있어요. 하지만 FBCSP는 **엄밀히 더 표현력 큰 방법**이고, 다른 피험자·데이터셋에선 이길 수 있습니다. **더 정교한 특징공학을 구현·시연**하는 의미입니다."

## 🎬 Bonus · Implementations
**📋** 얼마나 깊이 — **자작 고유분해/SVD/공분산? 복잡한 차원축소? 튜닝/학습? 자작 분류기?**

### A. 자작 고유분해 (Jacobi) — "own eigendecomposition"
**🗣️** "`eigh`마저 라이브러리 없이 **순수 numpy로 cyclic Jacobi** 알고리즘을 짰습니다(`jacobi.py`)."
**🗣️ *어떻게* 도나 (깊이):** "행렬의 **비대각 원소를 한 쌍씩 회전으로 0**으로 만들길 반복(sweep)합니다. 비대각이 충분히 작아지면 멈추고 — **남은 대각이 고유값**, 누적한 회전이 **고유벡터**입니다. 일반화 고유분해 `C1 w = λ(C1+C2)w`는 **백색화(whitening)**로 표준 문제로 바꾼 뒤 Jacobi를 두 번 적용. 쓰는 연산은 `+,*,sqrt,sign,matmul`뿐 — `np.linalg.eig/eigh/svd`나 scipy 안 부름."
**💻**
```
[A] Jacobi 정확도            : 0.8444  (scipy eigh와 동일)
    표준 고유분해 vs numpy   : 5.33e-14   ← 기계 정밀도
    일반화 고유분해 vs scipy : 1.55e-15   ← 사실상 완벽
```
**🗣️** "고유분해는 보통 블랙박스 라이브러리 호출인데, 밑바닥부터 짜서도 5e-14로 일치 — 핵심 선형대수를 이해했다는 증거." `MyCSP(solver="jacobi")`로 켭니다.

### C. 하이퍼파라미터 튜닝 — "tuning/learning"
**🗣️** "`GridSearchCV`로 `n_components ∈ {4,6,8}`을 탐색(`evaluate.tune`). **inner fold마다 CSP 재학습**해 **누수 없는 중첩 교차검증**."
**💻** `[C] tuned {'csp__n_components': 8} -> cv 0.8444`
**🗣️ 깊이:** "모델 선택 + 중첩 CV 누수 방지를 이해한 부분. run_all(공식)은 비교 공정성 위해 **고정 4성분** 유지, 튜닝은 데모."

### D. 자작 분류기 (OwnLDA) — "own classifier"
**🗣️** "LDA도 직접(`own_lda.py`). 풀드 공분산 → **`w = Σ⁻¹(μ1−μ0)`**, 절편은 사전확률 보정, 판정 `w·x+b` 부호. `np.linalg.solve`만."
**💻**
```
[D] OwnLDA 정확도          : 0.7889
    sklearn LDA 예측 일치율 : 100.0%   ← 완전 동일
```
**🗣️ 깊이:** "분류기를 `LDA()` 호출이 아니라 **수식으로 직접** 구현해 100% 일치 — 분류기 수학을 이해했다는 증거." `build_pipeline(clf="own-lda")`로 켭니다.

### (+ F. FBCSP = "복잡한 차원축소"에도 해당)

- *일반화 고유분해 방법?* "백색화로 표준 문제화 후 Jacobi 두 번."
