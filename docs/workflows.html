Total Perspective Vortex — Workflows

프로젝트의 전체 동작·구조·평가 흐름을 9개 다이어그램으로. (브라우저에서 자동 렌더링)

흐름도

1. 전체 파이프라인 2. 모듈 구조 3. Train (A/B) 4. Predict / Realtime 5. Score 60% 게이트 6. CSP 내부 7. 교차검증 + 누수 8. 개념 9단계 9. 보너스 매핑

1전체 파이프라인 (raw 신호 → 점수)

원본 EEG가 점수가 되기까지의 큰 흐름.

2모듈 구조 (어떤 파일이 무엇을 부르나)

CLI 입구부터의 호출 관계. 점선은 보너스 경로.

3Train — 두 갈래 (A 채점 / B 저장)

A = 정직한 점수(게이트가 쓰는 값, 모델은 버림). B = predict가 굴릴 실제 모델 + 시험지 9개.

4Predict / Realtime — 스트림 시뮬레이션

저장된 모델로 처음 보는 9문제를 하나씩 예측 (이벤트당 2초 미만).

5Score — 60% 게이트 (run_all)

109명 × 6실험 → 실험 유형별 평균(6개) → 그 6개의 평균.

6CSP 내부 (fit + transform)

공분산 → 고유분해 → 상위 4 필터 → 로그-분산. lambda=a/(a+b)가 0.5에서 멀수록 좋음. 보너스로 eigh를 자작 Jacobi로 교체 가능(scipy와 ~1e-14 일치).

7교차검증 + 누수 방지 (한 fold 안)

CSP가 Pipeline 첫 단계라 fold마다 학습분으로만 재학습 → 누수 없음 (누수 시 1.0 vs 정상 0.844).

8개념 9단계 (배움의 지도)

평가·이해를 위한 개념 순서.

9보너스 5개 → 평가표 항목 매핑

구현한 보너스가 어느 채점 항목에 해당하는지.
